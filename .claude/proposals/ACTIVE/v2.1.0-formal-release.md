# v2.1.0 — Formal Release Bundle

## Scope

The first formal `v2.1.0` tag bundles four streams of work that had
each been lingering in their own ACTIVE proposal:

1. **Engine bug bundle** — six verified bugs surfaced in the demo
   session, scoped down from ten reports after independent verification
   ruled out three "phantom" reports against stale code and one
   intentional design choice (rc19).
2. **CI/CD pipeline** (absorbs `v2.0.5-cicd-pipeline.md`) —
   re-scoped to free GitHub-hosted runners only after CUDA OOMs proved
   uncopable. Releases are locally hand-published.
3. **Python wrapper** (absorbs `v2.1.1-python-wrapper.md`) —
   playwright pattern: a ~50 KB pure-Python wheel + on-demand engine
   download via `entropic install-engine`. No PyPI exemption, no
   wheel index.
4. **OpenAI-compat example** (absorbs `v2.0.7-openai-compat-example.md`) —
   demonstrates the C ABI is sufficient for an HTTP front-end serving
   the OpenAI Chat Completions schema.

The `entropic-2x-release-primer.md` repo-metadata work folds into the
same release commit (description, topics, README, version bump).

  `tasks_mutex_` and captures it as `attached_gen_`; the cb's first
  action is `observer_call_is_stale()`; `cancel_inflight_async_tasks`
  bumps the generation unconditionally after its 1 s poll. New SCENARIO
  in `external_bridge_test.cpp` covers attach/detach + a stale-thread
  simulation.

## Out of scope (explicit)

- macOS / Windows / aarch64 / NPU — roadmap-level.
- Self-hosted CUDA runner — declined; releases stay locally published.
- PyPI fat-wheel exemption — avoided via the playwright pattern.
- Removing `RoutingConfig` entirely — deprecated only; full removal in
  v2.2.0.
- Embeddings, fine-tuning, image, audio, assistants endpoints in the
  OpenAI example.
- Engine API changes beyond the bug fixes — interface headers
  (`include/entropic/interfaces/i_*.h`) untouched.
