## Scope

Core engine library: `librentropic-core.so`. After v1.8.4, the engine can
accept a user prompt, route to a tier, generate a response via the inference
interface contract, manage context (compaction, anchors, pruning, warnings),
process directives from tool results, and track loop metrics. Interrupt and
pause are thread-safe via `std::atomic`.

This is a faithful port of:
- `src/entropic/core/engine.py` -- `AgentEngine`: main loop, state machine, directive handlers, interrupt/pause, auto-chain
- `src/entropic/core/engine_types.py` -- `AgentState` (10 states), `LoopConfig`, `LoopContext`, `LoopMetrics`, `EngineCallbacks` (15 callbacks), `GenerationEvents`, `InterruptContext`, `InterruptMode`, `ToolApproval`
- `src/entropic/core/response_generator.py` -- `ResponseGenerator`: tier routing, system prompt assembly, streaming with interrupt/pause, tool filtering per tier
- `src/entropic/core/context_manager.py` -- `ContextManager`: compaction trigger, tool result pruning, context anchors, context warning injection
- `src/entropic/core/directives.py` -- `DirectiveProcessor`: registry-based dispatcher, 11 directive types, wire format extraction
- `src/entropic/core/compaction.py` -- `CompactionManager`: value-density compaction, structured summary extraction (no model inference), `TokenCounter`

v1.8.2 provided inference backend, orchestrator, chat adapters, and
generation. v1.8.3 provided KV cache prefix caching. v1.8.4 builds the
operational heart of the engine: the agentic loop that drives generation
and manages state.

**What this version does NOT include:**
- Tool execution (v1.8.5) -- `ToolExecutor`, `MCPServerBase`, `ToolRegistry`
- Delegation/pipeline execution (v1.8.6) -- `DelegationManager`, worktrees
- External MCP (v1.8.7) -- stdio/SSE transport
- Session persistence (v1.8.8) -- SQLite storage

Without tools, the loop can still: generate responses, route tiers, manage
context, trigger compaction, process directives (from stubbed tool results
in tests), and fire all callbacks. Delegation directive handlers store
pending requests in `LoopContext.metadata` but do not execute them (that
requires `DelegationManager` from v1.8.6).

**Core has ZERO dependencies** on inference, MCP, storage, or config
libraries. It communicates through interface contracts only:
- Inference: `IInferenceCallbacks` (C function pointers for generate/stream)
- Config: structs passed by value at construction time
- Storage: not used (v1.8.8)

## State Machine

### AgentState Enum (10 states)

```
IDLE           Initial state before run()
PLANNING       Entered at run() start, before first generation
EXECUTING      Active generation in progress
WAITING_TOOL   Waiting for tool execution (v1.8.5)
VERIFYING      Post-tool verification (v1.8.5)
DELEGATING     Child delegation in progress (v1.8.6)
COMPLETE       Loop terminated successfully
ERROR          Loop terminated due to repeated errors
INTERRUPTED    Loop terminated by user interrupt
PAUSED         Generation paused, awaiting user input
```

### State Transition Diagram

```
                    ┌──────────────────────────────────────────┐
                    │                                          │
                    v                                          │
  IDLE ──> PLANNING ──> EXECUTING ──> COMPLETE                │
                           │  ^                                │
                           │  │ (next iteration)               │
                           │  └────────────────┐               │
                           │                   │               │
                           ├──> WAITING_TOOL ──┘  (v1.8.5)     │
                           │                                   │
                           ├──> DELEGATING ────┘  (v1.8.6)     │
                           │                                   │
                           ├──> PAUSED ────────┘               │
                           │                                   │
                           ├──> INTERRUPTED                    │
                           │                                   │
                           └──> ERROR                          │
                                                               │
  Any state ──> INTERRUPTED  (via atomic flag)                 │
  PAUSED ──> EXECUTING  (resume)                               │
  PAUSED ──> INTERRUPTED  (cancel_pause)                       │
```

### Termination Conditions

1. `ctx.state == COMPLETE` -- model response complete, no tool calls, `is_response_complete()` true
2. `ctx.state == ERROR` -- `consecutive_errors >= max_consecutive_errors`
3. `ctx.state == INTERRUPTED` -- user set interrupt flag
4. `ctx.metrics.iterations >= max_iterations` -- safety limit
5. `ctx.consecutive_duplicate_attempts >= 3` -- stuck model circuit breaker

## Loop Algorithm

The main loop is synchronous in C++ (no async/await). Streaming is callback-based
via function pointers. The algorithm:

```
entropic_run(engine, messages_json, params_json):
    ctx = LoopContext{}
    ctx.metrics.start_time = now()
    build_initial_messages(ctx, messages_json)
    reinject_context_anchors(ctx)
    set_state(ctx, PLANNING)

    while !should_stop(ctx):
        ctx.metrics.iterations++

        if interrupt_flag.load():
            set_state(ctx, INTERRUPTED)
            break

        execute_iteration(ctx)

        if ctx.state == ERROR:
            break

    ctx.metrics.end_time = now()
    return serialize_result(ctx)
```

### execute_iteration(ctx)

```
1. refresh_context_limit(ctx)          // Update token budget if tier changed
2. prune_old_tool_results(ctx)         // TTL-based auto-prune
3. check_compaction(ctx)               // Compact if over threshold
4. set_state(ctx, EXECUTING)
5. [content, tool_calls] = generate_response(ctx)  // Via inference interface
6. append assistant message to ctx.messages
7. fire on_message callback
8. IF tool_calls:
      (v1.8.5 processes them — in v1.8.4 this branch is dead code)
   ELSE:
      evaluate_no_tool_decision(ctx, content)
```

### evaluate_no_tool_decision(ctx, content)

```
IF finish_reason == "interrupted":  set_state(INTERRUPTED), return
IF finish_reason == "length":       return (continue loop for more output)
IF auto_chain fires:                tier change, return (continue loop)
IF explicit_completion && tool_calls > 0:  return (wait for entropic.complete)
IF adapter.is_response_complete(content):  set_state(COMPLETE)
```

### generate_response(ctx) (ResponseGenerator)

```
1. lock_tier_if_needed(ctx):
   a. IF ctx.locked_tier != null: notify callback, return
   b. route(ctx.messages) via inference interface → tier
   c. ctx.locked_tier = tier
   d. rebuild system prompt for routed tier's adapter + tool filter
   e. fire on_tier_selected, on_routing_complete callbacks

2. IF stream_output:
   a. call inference generate_streaming with cancel flag
   b. for each token:
      - check interrupt_flag → break
      - check pause_flag → handle_pause(ctx, partial)
      - append to content, fire on_stream_chunk
   c. parse tool calls from content via adapter

3. ELSE:
   a. call inference generate (batch)
   b. extract content + tool_calls from result
```

## Interface Contracts for Core

Core has zero compile-time dependencies on other Entropic libraries. It
communicates through contracts injected at construction time.

### Inference Contract (injected as function pointers)

```cpp
/// @brief Function pointer types for inference operations.
/// @version 1.8.4
///
/// Core does not #include any inference headers. These function pointer
/// types define the contract. The facade (librentropic.so) wires them
/// to the actual InferenceBackend at engine creation time.

/// @brief Generate a response (batch mode).
/// @param messages_json JSON array of messages.
/// @param params_json Generation parameters JSON.
/// @param result_json Output: JSON result string. Caller must free.
/// @param user_data Opaque pointer to backend instance.
/// @return 0 on success, error code on failure.
typedef int (*entropic_generate_fn)(
    const char* messages_json,
    const char* params_json,
    char** result_json,
    void* user_data);

/// @brief Generate a response with streaming.
/// @param messages_json JSON array of messages.
/// @param params_json Generation parameters JSON.
/// @param on_token Callback invoked per token.
/// @param token_user_data Opaque pointer for on_token callback.
/// @param cancel Pointer to atomic cancel flag.
/// @param user_data Opaque pointer to backend instance.
/// @return 0 on success, error code on failure.
typedef int (*entropic_generate_streaming_fn)(
    const char* messages_json,
    const char* params_json,
    void (*on_token)(const char* token, size_t len, void* user_data),
    void* token_user_data,
    int* cancel,
    void* user_data);

/// @brief Route messages to determine tier.
/// @param messages_json JSON array of messages.
/// @param result_json Output: JSON routing result. Caller must free.
/// @param user_data Opaque pointer to orchestrator.
/// @return 0 on success.
typedef int (*entropic_route_fn)(
    const char* messages_json,
    char** result_json,
    void* user_data);

/// @brief Raw text completion without chat template.
/// @param prompt Raw prompt string (no chat formatting).
/// @param params_json Generation parameters JSON.
/// @param result_json Output: JSON result string. Caller must free.
/// @param user_data Opaque pointer to backend instance.
/// @return 0 on success, error code on failure.
typedef int (*entropic_complete_fn)(
    const char* prompt,
    const char* params_json,
    char** result_json,
    void* user_data);

/// @brief Parse tool calls from raw model output.
/// @param raw_content Raw model output string.
/// @param cleaned_content Output: cleaned content. Caller must free.
/// @param tool_calls_json Output: JSON array of tool calls. Caller must free.
/// @param user_data Opaque pointer to adapter.
/// @return 0 on success.
typedef int (*entropic_parse_tool_calls_fn)(
    const char* raw_content,
    char** cleaned_content,
    char** tool_calls_json,
    void* user_data);
```

**Location:** These typedefs belong in `include/entropic/interfaces/i_inference_callbacks.h`,
NOT in `core/engine.h`. The engine depends on the interface contract, not on the
inference library. The facade wires the concrete implementations at engine creation time.

## Implementation Plan

### Phase 1: Types and Structures

Extend `include/entropic/types/` with any types not already present from
v1.8.0. Most types (`AgentState`, `DirectiveType`, `Message`, `ToolCall`)
were established in v1.8.0. This phase adds:

- `LoopConfig`, `LoopMetrics`, `LoopContext` structs
- `EngineCallbacks` struct (C function pointers + user_data)
- `InterruptMode`, `ToolApproval` enum classes
- `InterruptContext` struct
- `GenerationEvents` struct (`std::atomic<bool>*` for interrupt and pause)

**Files:**
- `include/entropic/core/engine_types.h` (new)

### Phase 2: TokenCounter and CompactionManager

Port `compaction.py`. Deterministic structured extraction with no model
inference. The `TokenCounter` uses the same ~4 chars/token heuristic as
Python.

```
CompactionManager::check_and_compact(messages, force)
    1. Count tokens via TokenCounter
    2. If below threshold and !force: return unchanged
    3. If compaction disabled: warn, return unchanged
    4. Classify messages by value density:
       - System message: always kept
       - User messages (source=user): always kept
       - Tool results, injections, warnings: stripped
       - Assistant messages: keep only most recent
    5. Build structured summary:
       - Original task (first user message)
       - Tool call log (name + brief result per call)
       - Files touched (read vs modified)
    6. Assemble: [system, summary_msg, user_msgs..., last_assistant]
    7. Return compacted messages + CompactionResult
```

**Files:**
- `include/entropic/core/compaction.h` (new)
- `src/core/compaction.cpp` (new)

### Phase 3: DirectiveProcessor

Port `directives.py`. Registry-based dispatcher. Directive extraction
from tool result JSON happens in the MCP layer (which has nlohmann/json),
NOT in core. The MCP layer parses the ServerResponse JSON envelope,
extracts the `directives` array into typed `Directive` structs (defined
in `types/directive.h`), and passes those structs to core's
`DirectiveProcessor`. Core never touches JSON — it receives typed C++
objects only.

```
DirectiveProcessor:
    register(DirectiveType, handler_fn)
    process(ctx, directives) -> DirectiveResult

extract_directives(tool_result_json) -> vector<Directive>
    1. Parse JSON
    2. Look for "_directives" array
    3. Deserialize each via type string -> DirectiveType enum -> struct
    4. Return typed directive list

deserialize_directive(json_obj) -> Directive
    1. Read "type" string
    2. Map to DirectiveType enum via static registry
    3. Read "params" object fields into struct
```

Handler function signature:
```cpp
/// @brief Directive handler function type.
/// @param ctx Current loop context (mutable).
/// @param directive The directive to process.
/// @param result Aggregate result (accumulator).
/// @version 1.8.4
using DirectiveHandler = std::function<void(
    LoopContext& ctx,
    const Directive& directive,
    DirectiveResult& result)>;
```

**Files:**
- `include/entropic/core/directives.h` (new)
- `src/core/directives.cpp` (new)

### Phase 4: ContextManager

Port `context_manager.py`. Handles context limit refresh, tool result
pruning, context warning injection, and compaction delegation.

```
ContextManager:
    refresh_context_limit(ctx)       // Update TokenCounter max from tier config
    prune_tool_results(ctx, keep_recent) -> (pruned_count, freed_chars)
    prune_old_tool_results(ctx)      // TTL-based auto-prune
    inject_context_warning(ctx)      // Warn at threshold% capacity
    check_compaction(ctx, force)     // Delegate to CompactionManager
```

ContextManager receives CompactionManager by reference and config values
at construction time. No compile-time dependency on config library.

**Files:**
- `include/entropic/core/context_manager.h` (new)
- `src/core/context_manager.cpp` (new)

### Phase 5: ResponseGenerator

Port `response_generator.py`. System prompt assembly, tier routing via
inference interface, streaming with interrupt/pause.

```
ResponseGenerator:
    generate_response(ctx) -> (content, tool_calls)
        1. lock_tier_if_needed(ctx)
        2. IF stream: generate_streaming(ctx)
           ELSE: generate_non_streaming(ctx)

    lock_tier_if_needed(ctx):
        IF ctx.locked_tier not empty: fire callback, return
        Call inference_interface.route(messages) -> tier
        ctx.locked_tier = tier
        Rebuild system prompt for tier's adapter + tool filter
        Fire on_tier_selected, on_routing_complete

    generate_streaming(ctx):
        Invoke inference_interface.generate_stream with cancel flag
        Per token: check interrupt, check pause, accumulate, fire callback
        Parse tool calls from accumulated content
        Return (cleaned_content, tool_calls)

    generate_non_streaming(ctx):
        Invoke inference_interface.generate
        Parse result JSON
        Return (content, tool_calls)
```

The `handle_pause` flow:
1. Save partial content into InterruptContext
2. Set state to PAUSED, fire on_state_change
3. Call on_pause_prompt callback (blocking -- consumer provides injection)
4. If injection is null: set interrupt, return partial
5. If injection is empty: resume, return partial
6. If injection has content: append partial + injection to messages,
   restart generation from new context

**Files:**
- `include/entropic/core/response_generator.h` (new)
- `src/core/response_generator.cpp` (new)

### Phase 6: AgentEngine

Port `engine.py`. The main engine class. Owns all subsystems, registers
directive handlers, implements the loop.

```
AgentEngine:
    // Construction
    AgentEngine(InferenceInterface, LoopConfig, CompactionConfig)

    // Public API (called via C API wrappers)
    run(messages_json, params_json) -> result_json
    run_streaming(messages_json, params_json, on_token, cancel) -> error_t
    interrupt()           // Set atomic interrupt flag
    reset_interrupt()     // Clear interrupt flag
    pause()              // Set atomic pause flag
    cancel_pause()       // Clear pause + set interrupt
    set_callbacks(EngineCallbacks)
    context_usage(messages) -> (used, max)

    // Internal
    _loop(ctx)
    _execute_iteration(ctx)
    _evaluate_no_tool_decision(ctx, content, tools_attempted)
    _should_stop(ctx) -> bool
    _set_state(ctx, state)
    _reinject_context_anchors(ctx)

    // Directive handlers (registered in constructor)
    _directive_stop_processing(ctx, directive, result)
    _directive_tier_change(ctx, directive, result)
    _directive_delegate(ctx, directive, result)      // Store pending, don't execute
    _directive_pipeline(ctx, directive, result)       // Store pending, don't execute
    _directive_complete(ctx, directive, result)
    _directive_clear_self_todos(ctx, directive, result) // No-op
    _directive_inject_context(ctx, directive, result)
    _directive_prune_messages(ctx, directive, result)
    _directive_context_anchor(ctx, directive, result)
    _directive_phase_change(ctx, directive, result)   // Validate phase exists
    _directive_notify_presenter(ctx, directive, result)
```

Interrupt mechanism:
- `std::atomic<bool> interrupt_flag_` -- checked at loop top + every streaming token
- `std::atomic<bool> pause_flag_` -- checked every streaming token
- `interrupt()` sets interrupt_flag_, clears pause_flag_
- `pause()` sets pause_flag_
- `cancel_pause()` clears pause_flag_, sets interrupt_flag_
- Thread-safe: UI thread calls interrupt/pause, generation thread reads atomics

**Files:**
- `include/entropic/core/engine.h` (new)
- `src/core/engine.cpp` (new)

### Phase 7: C API Implementation

Implement the `entropic_run()` and `entropic_run_streaming()` functions
that were stubbed in v1.8.0's `entropic.h`. These are the public entry
points in `librentropic.so` (facade) that create an `AgentEngine`, wire
the inference interface, and run the loop.

```c
/// @brief Run the engine synchronously.
/// @param engine Engine handle.
/// @param messages_json JSON array of input messages.
/// @param params_json Engine parameters JSON (LoopConfig fields).
/// @param result_json Output: JSON result. Caller must free.
/// @return ENTROPIC_OK on success.
/// @version 1.8.4
ENTROPIC_EXPORT entropic_error_t entropic_run(
    entropic_engine_t engine,
    const char* messages_json,
    const char* params_json,
    char** result_json);

/// @brief Run the engine with streaming output.
/// @param engine Engine handle.
/// @param messages_json JSON array of input messages.
/// @param params_json Engine parameters JSON.
/// @param on_token Per-token callback.
/// @param user_data Opaque pointer for callback.
/// @param cancel Atomic cancel flag (set to non-zero to abort).
/// @return ENTROPIC_OK on success.
/// @version 1.8.4
ENTROPIC_EXPORT entropic_error_t entropic_run_streaming(
    entropic_engine_t engine,
    const char* messages_json,
    const char* params_json,
    void (*on_token)(const char* token, size_t len, void* user_data),
    void* user_data,
    int* cancel);
```

The facade wires InferenceInterface by capturing the loaded backend and
orchestrator handles into the function pointers. Core never knows about
the concrete types.

## Validation Criteria

1. **Headless test passes**: prompt in, response out, correct state transitions
2. **State machine correct**: all transitions match diagram, no illegal transitions
3. **Interrupt works**: setting atomic flag stops generation within one token
4. **Pause works**: pause flag triggers callback, injection continues generation
5. **Compaction works**: messages over threshold are compacted, token count drops
6. **Context anchors survive compaction**: anchors reinjected after compact
7. **Directives parse correctly**: JSON wire format -> typed structs -> handler dispatch
8. **All 11 directive handlers fire**: unit test per handler type
9. **Metrics accurate**: iteration count, duration, token count correct
10. **Termination conditions**: max_iterations, max_errors, duplicate detection all stop the loop
11. **No inference dependency**: core compiles and links without inference headers
12. **C API works**: entropic_run returns valid JSON result through the facade
13. **SLOC <= 50 per function**: all functions pass knots complexity gates
14. **Doxygen complete**: every public symbol has @brief and @version
