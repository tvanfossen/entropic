## Scope

Runtime identity lifecycle management. After v1.9.6, consumers can create,
update, and destroy identities programmatically without restarting the engine.
Dynamic identities are first-class -- they participate in tier routing, have
system prompts, grammar constraints, MCP key sets, LoRA adapter paths, phases,
and all other capabilities of static (YAML-file-loaded) identities.

This is the feature that separates Entropic-as-a-library from
Entropic-as-a-fixed-configuration. The three consumer archetypes each need it:

1. **Game engine**: NPCs created at runtime as the player enters new areas.
   Each NPC has a unique system prompt, personality, allowed tools (inventory
   manipulation, dialog, no filesystem), and optional LoRA adapter for
   fine-tuned behavior. NPCs are destroyed when the player leaves the area
   to reclaim the routing slot.

2. **Homeschool**: Tutoring personas created per subject/session. A "math tutor"
   identity with a math-focused grammar constraint and read-only MCP keys for
   reference materials. A "writing coach" identity with different system prompt
   and no grammar constraint. Created by the parent/teacher orchestrator,
   destroyed at session end.

3. **TUI developer**: Less critical (static identities suffice), but enables
   ad-hoc identity creation for experimentation. "Create a security auditor
   identity with only read access to filesystem and git."

**Dependencies:**
- **v1.8.6** (Identity system): `IdentityFrontmatter`, `PhaseConfig`, `allowed_tools`,
  `auto_chain`, `role_type`, phases, delegation infrastructure. Dynamic identities
  reuse the same `IdentityConfig` struct and participate in the same routing/delegation
  system as static identities.
- **v1.9.3** (Grammar registry): `grammar_id` references named grammars from the
  `GrammarRegistry`. Dynamic identities reference grammars by key, not by file path.
  The grammar must already exist in the registry.
- **v1.9.4** (MCP authorization): `mcp_keys` references MCP key patterns from the
  `MCPAuthorizationManager`. Dynamic identities get a key set registered at creation
  time.

**What this version does NOT include:**
- Identity persistence across engine restarts (consumers serialize via C API if needed)
- Identity templates / inheritance (create from another identity as base)
- Identity migration (hot-swap system prompt without destroy/recreate)
- Dynamic routing weight adjustment (router retraining for new identities)

**What this version adds beyond the Python codebase:**
The Python engine has no dynamic identity creation. All identities are loaded
from `data/prompts/identity_*.md` files at startup. v1.9.6 is a new capability
that has no Python behavioral spec to port.

## IdentityConfig Struct

Dynamic identities are described by the same fields as static
`IdentityFrontmatter`, unified into a single C++ struct. Static identities
populate this from YAML frontmatter (v1.8.6). Dynamic identities populate
it from the C API.

```cpp
/// @file include/entropic/config/identity.h (extended from v1.8.6)
/// @brief Identity configuration -- shared by static and dynamic identities.
/// @version 1.9.6

namespace entropic {

/// @brief Origin of an identity -- how it was created.
/// @version 1.9.6
enum class IdentityOrigin : uint8_t {
    STATIC  = 0,    ///< Loaded from YAML frontmatter file at startup
    DYNAMIC = 1,    ///< Created at runtime via API
};

/// @brief Full identity configuration.
/// @version 1.9.6 (extended from v1.8.6 IdentityFrontmatter)
///
/// All fields from IdentityFrontmatter are present. Dynamic identities
/// set these programmatically; static identities set them from YAML parsing.
///
/// The `origin` field distinguishes static from dynamic identities. This
/// matters for lifecycle: static identities cannot be destroyed via the
/// dynamic identity API (they are owned by the config loader).
struct IdentityConfig {
    std::string name;                           ///< Unique identity name (e.g., "eng", "npc_blacksmith")
    std::string system_prompt;                  ///< Full system prompt text (markdown body)
    std::vector<std::string> focus;             ///< Classification focus keywords (min 1)
    std::vector<std::string> examples;          ///< Few-shot classification examples
    std::string grammar_id;                     ///< Grammar registry key (empty = no grammar)
    std::string auto_chain;                     ///< Auto-chain target tier name (empty = none)
    std::vector<std::string> allowed_tools;     ///< Tool filter list (empty = all tools via identity)
    std::vector<std::string> bash_commands;     ///< Allowed bash commands
    std::vector<MCPKey> mcp_keys;               ///< MCP authorization keys (v1.9.4)
    std::string adapter_path;                   ///< LoRA adapter path (v1.9.2, empty = base model)
    int max_output_tokens = 1024;               ///< Default max output tokens
    float temperature = 0.7f;                   ///< Default temperature
    float repeat_penalty = 1.1f;                ///< Default repeat penalty
    bool enable_thinking = false;               ///< Default enable_thinking
    std::string model_preference = "primary";   ///< Model key from registry
    bool interstitial = false;                  ///< Interstitial (non-routable, triggered by tools)
    bool routable = true;                       ///< Participates in tier routing
    std::string role_type = "front_office";     ///< "front_office", "back_office", "utility"
    bool explicit_completion = false;           ///< Requires explicit entropic.complete to finish
    std::unordered_map<std::string, PhaseConfig> phases;  ///< Named inference phases
    IdentityOrigin origin = IdentityOrigin::DYNAMIC;      ///< How this identity was created
};

} // namespace entropic
```

### Field Notes

**`name`**: Must be unique across all identities (static + dynamic). The
`IdentityManager` rejects creation if the name collides. Names are
case-sensitive, lowercase-only, alphanumeric plus underscore/hyphen.
Regex: `^[a-z][a-z0-9_-]{0,63}$` (max 64 chars).

**`system_prompt`**: The markdown body that becomes the system message. For
static identities this is the content after the YAML frontmatter delimiter.
For dynamic identities the consumer provides it directly.

**`focus`**: At least one keyword is required. These appear in the classification
prompt (`build_classification_prompt()`) and determine routing behavior.

**`examples`**: Optional few-shot classification examples. Interleaved into the
classification prompt via `_interleave_examples()`.

**`grammar_id`**: References a key in the `GrammarRegistry` (v1.9.3). If
non-empty, the grammar must exist in the registry at creation time. Validation
fails otherwise. An empty string means no grammar constraint.

**`mcp_keys`**: If non-empty, a key set is registered with
`MCPAuthorizationManager` (v1.9.4) at creation time. Empty means no
per-identity enforcement (layers 1+2 still apply).

**`adapter_path`**: Optional LoRA adapter file path (v1.9.2). If specified,
the `AdapterManager` loads the adapter on first use. Empty means base model.

## IdentityManager Class

Central manager for all identities (static and dynamic). Replaces the current
pattern where identities are loaded ad-hoc by the config loader and stored in
disparate places across the orchestrator and engine.

```cpp
/// @file include/entropic/core/identity_manager.h
/// @brief IdentityManager -- lifecycle management for static and dynamic identities.
/// @version 1.9.6
///
/// @par Responsibilities:
/// - Own all IdentityConfig instances (static and dynamic)
/// - Enforce max_identities limit
/// - Validate identity configs at creation time
/// - Coordinate with GrammarRegistry, MCPAuthorizationManager, AdapterManager
/// - Rebuild router classification prompt when identities change
/// - Provide identity lookup by name for engine, orchestrator, delegation
///
/// @par Thread safety:
/// - get() and has() are lock-free (read from snapshot via shared_mutex)
/// - create/update/destroy acquire exclusive lock on identities_mutex_
/// - Router prompt rebuild is deferred (flagged dirty, rebuilt on next classify)
///
/// @par Ownership:
/// Owned by AgentEngine. One IdentityManager per engine instance. Replaces
/// the scattered identity storage across ConfigLoader and ModelOrchestrator.

#pragma once

#include <entropic/config/identity.h>
#include <entropic/types/error.h>

#include <shared_mutex>
#include <string>
#include <unordered_map>
#include <vector>

namespace entropic {

// Forward declarations
class GrammarRegistry;
class MCPAuthorizationManager;

/// @brief Configuration for the identity manager.
/// @version 1.9.6
struct IdentityManagerConfig {
    size_t max_identities = 64;     ///< Maximum total identities (static + dynamic)
    bool allow_dynamic = true;      ///< Master toggle for dynamic identity creation
};

/// @brief Manages the lifecycle of all identities in the engine.
/// @version 1.9.6
///
/// @par Lifecycle:
/// @code
///   IdentityManager mgr(config, grammar_reg, auth_mgr);
///   mgr.load_static(config_loader);        // Load from YAML files
///   mgr.create("npc_guard", {...});         // Dynamic creation
///   auto* id = mgr.get("npc_guard");       // Lookup
///   mgr.update("npc_guard", {...});         // Modify
///   mgr.destroy("npc_guard");              // Remove
/// @endcode
class IdentityManager {
public:
    /// @brief Construct with dependencies.
    /// @param config Identity manager configuration.
    /// @param grammar_registry Reference for grammar_id validation.
    /// @param auth_manager Reference for MCP key set registration.
    /// @version 1.9.6
    IdentityManager(const IdentityManagerConfig& config,
                    GrammarRegistry& grammar_registry,
                    MCPAuthorizationManager& auth_manager);

    /// @brief Load static identities from config loader.
    /// @param identities Vector of static identity configs.
    /// @return Number of identities loaded.
    /// @version 1.9.6
    ///
    /// Called once at engine startup. Static identities have
    /// origin=STATIC and cannot be destroyed via destroy().
    size_t load_static(const std::vector<IdentityConfig>& identities);

    /// @brief Create a new dynamic identity.
    /// @param config Identity configuration.
    /// @return ENTROPIC_OK on success.
    ///         ENTROPIC_ERROR_INVALID_CONFIG if validation fails.
    ///         ENTROPIC_ERROR_LIMIT_REACHED if max_identities exceeded.
    ///         ENTROPIC_ERROR_ALREADY_EXISTS if name already taken.
    /// @version 1.9.6
    ///
    /// Validates:
    /// - Name format (regex, uniqueness)
    /// - focus is non-empty
    /// - grammar_id exists in GrammarRegistry (if non-empty)
    /// - adapter_path is accessible (if non-empty)
    /// - max_identities not exceeded
    ///
    /// Side effects on success:
    /// - Registers MCP key set with MCPAuthorizationManager (if mcp_keys non-empty)
    /// - Flags router classification prompt as dirty
    /// - Logs at INFO: "Dynamic identity created: {name}"
    entropic_error_t create(const IdentityConfig& config);

    /// @brief Update an existing dynamic identity.
    /// @param name Identity name to update.
    /// @param config New configuration (name field must match).
    /// @return ENTROPIC_OK on success.
    ///         ENTROPIC_ERROR_NOT_FOUND if identity doesn't exist.
    ///         ENTROPIC_ERROR_PERMISSION_DENIED if identity is static.
    ///         ENTROPIC_ERROR_INVALID_CONFIG if validation fails.
    /// @version 1.9.6
    ///
    /// Replaces the entire config. Partial updates are not supported --
    /// the consumer reads the current config, modifies fields, and passes
    /// the full config back.
    ///
    /// Side effects on success:
    /// - Updates MCP key set (unregister old, register new)
    /// - Flags router classification prompt as dirty (if focus/examples changed)
    /// - Logs at INFO: "Dynamic identity updated: {name}"
    entropic_error_t update(const std::string& name,
                            const IdentityConfig& config);

    /// @brief Destroy a dynamic identity.
    /// @param name Identity name to destroy.
    /// @return ENTROPIC_OK on success.
    ///         ENTROPIC_ERROR_NOT_FOUND if identity doesn't exist.
    ///         ENTROPIC_ERROR_PERMISSION_DENIED if identity is static.
    ///         ENTROPIC_ERROR_IN_USE if identity is currently active in
    ///         a delegation (locked_tier matches name).
    /// @version 1.9.6
    ///
    /// Side effects on success:
    /// - Unregisters MCP key set from MCPAuthorizationManager
    /// - Flags router classification prompt as dirty
    /// - Logs at INFO: "Dynamic identity destroyed: {name}"
    entropic_error_t destroy(const std::string& name);

    /// @brief Get identity config by name.
    /// @param name Identity name.
    /// @return Pointer to config, or nullptr if not found.
    /// @version 1.9.6
    ///
    /// The returned pointer is valid until the identity is destroyed
    /// or updated. Lock-free read via shared_mutex.
    const IdentityConfig* get(const std::string& name) const;

    /// @brief Check if an identity exists.
    /// @param name Identity name.
    /// @return true if identity exists.
    /// @version 1.9.6
    bool has(const std::string& name) const;

    /// @brief List all identity names.
    /// @return Vector of identity names (static + dynamic).
    /// @version 1.9.6
    std::vector<std::string> list() const;

    /// @brief List only routable identities (for classification prompt).
    /// @return Vector of pointers to routable identity configs.
    /// @version 1.9.6
    ///
    /// Routable identities have routable=true and are included in the
    /// classification prompt. Non-routable identities (interstitial,
    /// utility) are excluded.
    std::vector<const IdentityConfig*> list_routable() const;

    /// @brief Get the number of identities.
    /// @return Total count (static + dynamic).
    /// @version 1.9.6
    size_t count() const;

    /// @brief Get the number of dynamic identities.
    /// @return Dynamic identity count only.
    /// @version 1.9.6
    size_t count_dynamic() const;

    /// @brief Check if the router classification prompt needs rebuilding.
    /// @return true if create/update/destroy has been called since last reset.
    /// @version 1.9.6
    bool is_router_dirty() const;

    /// @brief Clear the dirty flag (called after router prompt rebuild).
    /// @version 1.9.6
    void clear_router_dirty();

private:
    IdentityManagerConfig config_;
    GrammarRegistry& grammar_registry_;
    MCPAuthorizationManager& auth_manager_;

    /// @brief All identities keyed by name.
    std::unordered_map<std::string, IdentityConfig> identities_;

    /// @brief Guards mutations to identities_ map.
    mutable std::shared_mutex identities_mutex_;

    /// @brief Flag: router classification prompt needs rebuilding.
    std::atomic<bool> router_dirty_{false};

    /// @brief Validate an identity config.
    /// @param config Config to validate.
    /// @param is_update true if this is an update (name must already exist).
    /// @return Empty string on success, error message on failure.
    /// @version 1.9.6
    std::string validate(const IdentityConfig& config,
                         bool is_update) const;

    /// @brief Validate identity name format.
    /// @param name Name to validate.
    /// @return Empty string on success, error message on failure.
    /// @version 1.9.6
    std::string validate_name(const std::string& name) const;

    /// @brief Register MCP keys for an identity.
    /// @param config Identity config with mcp_keys.
    /// @version 1.9.6
    void register_mcp_keys(const IdentityConfig& config);

    /// @brief Unregister MCP keys for an identity.
    /// @param name Identity name.
    /// @version 1.9.6
    void unregister_mcp_keys(const std::string& name);
};

} // namespace entropic
```

## Router Integration

Dynamic identities participate in tier routing. The classification prompt
(`build_classification_prompt()`) must include dynamic identities alongside
static ones.

### Dirty Flag Pattern

Rebuilding the classification prompt on every classify call is wasteful.
Instead, `IdentityManager` flags the router as dirty when identities change.
The router checks the flag before classifying and rebuilds only when needed.

```
create_identity("npc_guard", ...)
    │
    IdentityManager:
    │ identities_["npc_guard"] = config
    │ router_dirty_ = true
    │
    ▼
Router::classify(message)
    │
    │ if (identity_manager_.is_router_dirty()) {
    │     routable = identity_manager_.list_routable();
    │     classification_prompt_ = build_classification_prompt(routable);
    │     identity_manager_.clear_router_dirty();
    │ }
    │
    │ prompt = format(classification_prompt_, message, history);
    │ digit = inference.complete(prompt);
    │ tier = routable[digit - 1];
    │
    ▼
    routes to "npc_guard" if classified
```

### Classification Prompt Rebuild

The digit-based classification system (1, 2, 3, ...) means dynamic identities
get digits assigned based on their position in the routable list. The list
order is:

1. Static identities first (preserving their original order from startup)
2. Dynamic identities appended in creation order

This means:
- Existing static identity digits are STABLE across dynamic identity changes
- Dynamic identity digits may shift when other dynamic identities are
  created/destroyed
- The small router model re-learns the new mapping implicitly (no retraining
  needed -- the classification prompt includes focus keywords and examples
  which guide the router)

The router model (Qwen3-0.6B) is a pure classifier with no persistent state.
Each classification call includes the full prompt with all tier descriptions
and examples. Changing the prompt changes routing behavior immediately.

## In-Use Protection

A dynamic identity cannot be destroyed while it is the `locked_tier` of any
active `LoopContext` (delegation in progress). The engine tracks active
delegations through the existing `DelegationManager` (v1.8.6).

```cpp
/// @brief Check if an identity is currently in use.
/// @param name Identity name.
/// @return true if any LoopContext has locked_tier == name.
/// @version 1.9.6
///
/// Called by destroy() before removing the identity. If in use,
/// destroy() returns ENTROPIC_ERROR_IN_USE.
bool is_in_use(const std::string& name) const;
```

The check queries the `DelegationManager` for active child contexts.
The lead loop's current tier is also checked. This prevents the scenario
where a consumer destroys an NPC identity mid-conversation.

Consumers who need to destroy an in-use identity must:
1. Wait for the delegation to complete
2. Or cancel the delegation first (`engine.interrupt()`)
3. Then destroy the identity

## C API

New functions in `entropic.h`. All cross the `.so` boundary as pure C --
identity configs are passed as JSON strings.

```c

ENTROPIC_ERROR_ALREADY_EXISTS,      ///< Identity name already taken
ENTROPIC_ERROR_IN_USE,              ///< Identity is active in delegation

/// @brief Create a dynamic identity.
/// @param handle Engine handle.
```

### Config JSON Wire Format

The JSON format mirrors `IdentityConfig` fields directly. JSON is the wire
format at the `.so` boundary; internally it deserializes to the C++ struct.

Fields not present in the JSON use defaults from `IdentityConfig`. This means
a minimal creation call needs only:

```json
{
    "name": "tutor_math",
    "system_prompt": "You are a patient math tutor for middle school students...",
    "focus": ["math", "algebra", "geometry", "arithmetic"]
}
```

All other fields default to their `IdentityConfig` initializer values.

### max_identities Enforcement

The `max_identities` limit applies to the TOTAL count of static + dynamic
identities. This prevents unbounded memory growth from dynamic creation.

Default: 64 (sufficient for 11 static + 53 dynamic). Configurable in YAML:

```yaml
# ~/.entropic/config.yaml or .entropic/config.local.yaml
engine:
  max_identities: 64
  allow_dynamic_identities: true
```

When the limit is reached, `create()` returns `ENTROPIC_ERROR_LIMIT_REACHED`.
The consumer must destroy unused identities before creating new ones.

Game consumers managing many NPCs should set this appropriately. The cost
per identity is the `IdentityConfig` struct (~2-4 KB depending on system
prompt length) plus the classification prompt rebuild cost (negligible for
the small router model).

## Validation Rules

Identity creation/update validation, executed in order:

```
validate(config):
    1. validate_name(config.name)
       - matches ^[a-z][a-z0-9_-]{0,63}$
       - not a reserved name ("default", "none", "all", "router")
       - for create: name not already in identities_ map
       - for update: name MUST already exist

    2. focus must be non-empty (at least one keyword)

    3. grammar_id (if non-empty):
       - grammar_registry_.has(config.grammar_id) must be true
       - "Grammar 'foo' not found in registry"

    4. auto_chain (if non-empty):
       - Must reference an identity that exists OR is being batch-created
       - Circular auto_chain detection is NOT performed (same as Python --
         engine loop depth limit prevents infinite chains)

    5. model_preference:
       - Must be a valid model key ("primary", "router", or custom)
       - Validation against bundled model registry

    6. role_type:
       - Must be "front_office", "back_office", or "utility"

    7. phases (if non-empty):
       - Each phase name must be non-empty
       - Phase temperature must be 0.0-2.0
       - Phase max_output_tokens must be > 0

    8. adapter_path (if non-empty):
       - File must exist and be readable
       - (actual LoRA load is deferred to first use)

    return "" on success, error message on first failure
```

## Validation Criteria

### V1: Create minimal identity

Create identity with only name, system_prompt, focus. Verify all defaults
applied. Verify `entropic_get_identity()` returns full config with defaults.

### V2: Create full identity

Create identity with all fields populated. Verify every field round-trips
through create + get.

### V3: Name format rejection

Attempt to create identities with invalid names: uppercase, spaces, leading
digit, empty, over 64 chars, reserved names ("default", "router"). All
return `ENTROPIC_ERROR_INVALID_CONFIG`.

### V4: Duplicate name rejection

Create "npc_guard". Attempt to create "npc_guard" again. Returns
`ENTROPIC_ERROR_ALREADY_EXISTS`.

### V5: max_identities enforcement

Set `max_identities: 3`. Load 2 static identities. Create 1 dynamic. Attempt
to create another. Returns `ENTROPIC_ERROR_LIMIT_REACHED`.

### V6: Grammar validation

Create identity with `grammar_id: "nonexistent"`. Returns
`ENTROPIC_ERROR_INVALID_CONFIG`. Create identity with a grammar_id that
exists in the registry. Succeeds.

### V7: Update replaces config

Create identity. Update with different system_prompt and temperature. Get
identity. Verify new values returned.

### V8: Update static identity rejected

Attempt to update a static identity (e.g., "eng"). Returns
`ENTROPIC_ERROR_PERMISSION_DENIED`.

### V9: Destroy dynamic identity

Create "npc_guard". Destroy "npc_guard". Verify `has("npc_guard")` returns
false. Verify count decreased.

### V10: Destroy static identity rejected

Attempt to destroy "eng". Returns `ENTROPIC_ERROR_PERMISSION_DENIED`.

### V11: Destroy in-use identity rejected

Create "npc_guard". Start delegation to "npc_guard". Attempt to destroy.
Returns `ENTROPIC_ERROR_IN_USE`. Complete delegation. Destroy succeeds.

### V12: Router dirty flag

Create identity. Verify `is_router_dirty()` returns true. Call
`clear_router_dirty()`. Verify false. Destroy identity. Verify true again.

### V13: Dynamic identity participates in routing

Create "npc_guard" with focus: ["guard", "security"]. Send message
"Who goes there?" to router. Verify classification includes "npc_guard"
as an option in the prompt.

### V14: Static identity digits stable

Load 3 static identities (digits 1, 2, 3). Create dynamic identity.
Static identities still have digits 1, 2, 3. Dynamic identity gets digit 4.
Destroy dynamic identity. Static identities still 1, 2, 3.

### V15: MCP keys registered on create

Create identity with mcp_keys. Verify `MCPAuthorizationManager::is_enforced()`
returns true. Verify `check_access()` works for the granted patterns.

### V16: MCP keys unregistered on destroy

Create identity with mcp_keys. Destroy identity. Verify
`MCPAuthorizationManager::is_enforced()` returns false.

### V17: MCP keys updated correctly

Create identity with READ keys. Update with WRITE keys. Verify new
access levels take effect.

### V18: list_identities returns all

Load 3 static, create 2 dynamic. `list()` returns 5 names. `count()`
returns 5. `count_dynamic()` returns 2.

### V19: list_routable excludes non-routable

Create identity with `routable: false`. Verify not in `list_routable()`.
Create identity with `routable: true`. Verify included.

### V20: C API round-trip

Call `entropic_create_identity()` with JSON. Call `entropic_get_identity()`.
Parse returned JSON. Verify all fields match.

### V21: C API list

Call `entropic_list_identities()`. Parse JSON array. Verify all identities
present.

### V22: C API error codes

Verify each error code (`LIMIT_REACHED`, `ALREADY_EXISTS`, `IN_USE`,
`NOT_FOUND`, `PERMISSION_DENIED`, `INVALID_CONFIG`) is returned for the
appropriate failure case through the C API.

### V23: Thread safety

Create/destroy identities from 4 concurrent threads. No data races
(ThreadSanitizer clean). All operations return correct results.

### V24: SLOC and complexity

All functions pass knots quality gates: cognitive complexity <= 15,
SLOC <= 50, returns <= 3.

### V25: Doxygen complete

Every public symbol in identity_manager.h has `@brief`, `@param`,
`@return`, `@version`.
