# Verification plan — v2.0.6 delegated-agent context bleed

Verifies both the mechanism (KV cache residue is actually cleared) and
the behavior (child delegations no longer leak prior-delegation content).

## Layer 1 — Mechanism (automated, fast)

Goal: prove that the prompt cache stores prefix-only KV and that cache
hits are actually being used (not silently falling back).

### 1a. Cache-hit validation via test log

Run the context-bleed regression:

```bash
build/full/tests/model/test-context-bleed
grep -E "Cache HIT|Cache MISS|Prompt cache|tok/s" \
  test-reports/model/logs/v2.0.6_delegation_context_bleed.log
```

**PASS criteria:**
- First generation: `Cache MISS` (expected — first time seeing this
  system prompt).
- Second generation with same system prompt: `Cache HIT: N tokens,
  M bytes` where N matches the prefix token count from pass 1.
- No `Cache restore failed, falling back to full prefill` warnings.
  Any such warning means the two-pass contract was violated and
  diagnosis is required.
- Throughput lines (`tok/s`) remain in the expected band (~17 tok/s
  for the IQ3_XXS primary model on reference hardware).

### 1b. Unit-level: `PromptCache` tests still pass

```bash
.venv/bin/inv test --filter prompt_cache
```

No PromptCache API change — the consumer's save/restore sequencing
changed, not the cache itself.

## Layer 2 — Behavior (automated, GPU-required)

### 2a. New regression test

```bash
.venv/bin/inv test --model --filter=context-bleed
```

Two scenarios:

| # | Gen A user | Gen B user | Assertion |
|---|---|---|---|
| 1 | "Invent a definition for QUIXOTIC_FLAMINGO_47" | "List the first three prime numbers" | Gen B content contains no "QUIXOTIC" or "FLAMINGO" |
| 2 | "Describe the TUI terminal interface" | "Did I ask anything previously? Yes/no and explain" | Gen B content contains no "TUI" or "terminal interface" |

Both use identical system prompts so gen B triggers the cache-hit
restore path. Without the fix, gen A's distinctive content bleeds via
residual KV; with the fix, gen B is clean.

**PASS criteria:** test passes; no `CHECK_FALSE` failures.

### 2b. Regression tests (mock-based, no GPU)

```bash
.venv/bin/inv test --filter regression
```

Should stay green — these don't exercise the real KV cache but they
validate that the delegation control flow wasn't disturbed by the fix.

**PASS criteria:**
- No `docs.search` tool calls with query "tui terminal interface
  text ui console" (or similar TUI-flavored queries) on turns 3 or 4
- Researcher/tracer child prompts (visible in `session.log`) contain
  no residual TUI content
- Diagnostic log lines appear showing `kv_bytes` proportional to
  `prefix_tokens` on cache hits

**FAIL criteria:** any child tool call on turns 3 or 4 mentioning TUI,
terminal, console, or other turn-2 vocabulary — or `kv_bytes` on
post-restore lines reflecting the prior prompt's full size.

## Layer 4 — Performance check (confirm no regression)

The fix adds one `llama_memory_seq_rm` call on the cache-hit path.
This is O(1) metadata — not a cache copy. Expected cost: microseconds.

```bash
# Before/after wall-clock on cache-hit path
.venv/bin/inv test --model --filter=context-retention
```

Watch the `Generated: N tokens, ... X tok/s` lines. Cache-hit-dominated
tests should show throughput within noise of pre-fix numbers (~26-36
tok/s per the proposal evidence).

## Failure triage

| Symptom | Likely cause | Action |
|---|---|---|
| Test 2a fails on marker assertion | Pass-1 prefill contains extra tokens beyond prefix, or `compute_prefix_token_count` is wrong | Check `run_prefill_cached` logs — first pass should prefill exactly `prefix_tokens`. If not, investigate `compute_prefix_token_count` + chat template application. |
| Test 2a fails on digit assertion for gen B | Model went off-topic for unrelated reasons | Review gen B raw content; tighten prompt if needed |
| `Cache restore failed, falling back to full prefill` warning | `state_seq_set_data` mismatch (stale cache entry, wrong model path in key) | Clear the cache (`unload → reload`) and retry; investigate key computation |
| Layer 3 still shows TUI bleed | Fix not exercised (stale .so linked) | `inv build` then `inv example --name=explorer`; confirm binary mtime |
| Throughput drops >5% on repeated same-system generations | Two-pass prefill amortized cost | Profile; pass 1 + pass 2 should cost the same as a single full prefill (same total tokens decoded) |

## Rollback

The v2.0.6 fix is isolated to `llama_cpp_backend.{cpp,h}`. To revert:

```bash
git diff src/inference/llama_cpp_backend.cpp \
         src/inference/llama_cpp_backend.h
# The key changes are:
#   - run_prefill_cached: replaces old single-pass miss path with
#     prefill_and_cache_prefix (two-pass)
#   - restore_cached_prefix: simplified (no seq_rm)
#   - new helper: decode_tokens_from
# Reverting any one of these independently will re-introduce the bleed.
# Revert all three as a unit.
```

The regression test in `tests/model/test_delegation_context_bleed.cpp`
stays — it's independent of the fix and guards against regression.
