## Summary

Delegated child agents (`researcher`, `tracer`) are issuing tool calls
that reference topics from **prior, unrelated delegations** in the same
session. The child's own prompt/message array is correctly scoped to
its current task, but its generated output leaks vocabulary from
previous tiers' work.

This is a quality regression that user-facing sessions expose. The
model wastes tool-call budget on irrelevant searches, and final
answers inherit off-topic material.

## Evidence

Session: `examples/explorer/.explorer/session.log` + `session_model.log`,
2026-04-15 14:35–14:54.

Four user turns during the session:

1. `tell me about the main engine loop in 3 minutes or less`
2. `Can you find any data for the TUI?`
3. `how simple would it be, define for steps, for me to copy paste this explorer example and run it with a 2B model instead?`
4. `uh you are running on examples/explorer right now, wouldnt it just be a change to default_config.yaml to have you use the secondary or lightweight model for the available roles instead?`

For turn 3, the parent lead tier correctly delegated to the researcher
with task: *"Find example files and configuration steps to run the
entropic explorer with a 2B model..."* — no TUI content.

The researcher's prompt at iteration 2, logged at 14:47:23, shows 7
messages:

```
[0] role=system [10387 chars]
[1] role=user  "Find example files and configuration steps to run the entropic explorer with a 2B model..."
[2] role=assistant  (empty — prior iter emitted tool calls only)
[3] role=user  "## Types — entropic::AdapterManager (class)... AgentEngine (class)..."  (tool result)
[4] role=user  "No files found matching 'examples/**/*'."  (tool result)
[5] role=user  "## Types — entropic::BackendInfo (struct)..."  (tool result)
[6] role=user  "No files found matching '*.md'."  (tool result)
```

**Zero TUI content in the child's context.** Yet the researcher then
emitted four parallel `docs.search` tool calls, one of which was:

```
<tool_call>
  <function=docs.search>
  <query>tui terminal interface text ui console</query>
  <limit>20</limit>
</tool_call>
```

The TUI search resolved in 5290 chars of noise, 8 ms of wasted inference,
and contaminated the researcher's next-iteration prompt with TUI
symbols that had nothing to do with the 2B-model task.

This is reproducible: turn 4 triggered the same pattern.

## Hypotheses (descending probability)

### 1. KV cache pollution across delegations

`src/inference/prompt_cache.cpp` keys cached KV state by prompt-prefix
hash. If a child delegation's system-prompt prefix shares tokens with
a prior tier's prompt, the cache may reuse residual KV state containing
tokens from the earlier delegation's content.

**Test:** add `logger->info("KV tokens entering child loop: {}",
llama_kv_self_used_cells(ctx))` at the top of the child-loop code in
`src/core/delegation.cpp`. If n_tokens > 0 when a child begins, the
backend is carrying state across tier boundaries.

**Fix shape:** explicit `llama_kv_self_clear(ctx)` between tier
transitions, or per-tier backend instances with isolated contexts.

### 2. System prompt includes recent-tier history

The routing / classification machinery may inject a "recent_tiers" or
"recent delegations" block into the system prompt. Turn 3's researcher
system prompt is 10387 chars — comparable to turn 2's but not
necessarily identical content.

**Test:** diff the 10387-char system prompts for turns 2 and 3
researcher invocations. If the system contains a rolling summary of
recent activity, TUI terms are plausibly baked in.

**Fix shape:** scope the recent-tier context to the current
delegation's session, or filter to same-tier history only.

### 3. Model exploration noise at temperature 0.70

MoE models with temperature 0.70 on batched tool-call generation can
produce off-topic queries. Lower probability — the TUI query is too
specific to be random.

**Test:** re-run turn 3 at temperature 0.0 (or very low) 5× and check
whether TUI searches still appear. If they disappear, it's noise.

## Scope

Investigation + diagnostic commit first, then fix. Do not skip straight
to a fix — the cause needs to be confirmed because all three hypotheses
demand different remediation.

### Phase 1: Diagnostic instrumentation

- Add `logger->info` at child-loop entry showing KV cache state
  (used-cells, prompt-prefix hash).
- Log the diff between consecutive same-tier system prompts.
- Re-run the explorer session, capture the TUI-bleed reproduction,
  inspect the new diagnostic output.

### Phase 2: Fix

Choose remediation based on Phase 1 findings. Draft decision rules:

| If Phase 1 reveals… | Remediate with… |
|---|---|
| KV cache used-cells > 0 at child entry | `llama_kv_self_clear` in child-engine init; audit the adapter swap path |
| System-prompt drift includes recent-tier content | Scope `recent_tiers` per-delegation, not session-wide |
| Clean state + deterministic reproduction at low temp | Accept as model noise; lower temperature for tool-call generation |

### Phase 3: Regression test

Add a regression test that:
- Spins up an agent with two tiers + docs.search mock.
- Issues delegation A on topic X; asserts child issues only X-related searches.
- Issues delegation B on topic Y; asserts child issues **only** Y-related searches (no X leakage).

Tests/unit/core/test_delegation_context_bleed.cpp, under the
regression label for CI.

## Out of scope

- Broader conversation-history pruning or context-compaction tuning.
- Multi-agent coordination features (see other proposals).

## Validation

Playback of the same 4-turn session that produced this report should
produce **zero** off-topic tool calls in child contexts.
