## Scope

LoRA adapter hot-swapping for the inference backend. After v1.9.2, the engine
can load a base model once and swap LoRA adapters without unloading or
reloading the base model. This enables per-identity fine-tuned behavior on a
single GPU with sub-200ms swap latency.

This builds on:
- v1.8.2 `InferenceBackend` / `LlamaCppBackend` -- base model lifecycle (COLD/WARM/ACTIVE)
- v1.8.2 `ModelOrchestrator` -- tier routing, model pool dedup, VRAM swap
- v1.9.1 `HookRegistry` -- `ON_ADAPTER_SWAP` hook point (defined, not yet wired)

llama.cpp provides native LoRA support:
- `llama_adapter_lora_init(model, path)` -- load adapter into RAM
- `llama_set_adapter_lora(ctx, adapter, scale)` -- activate on context
- `llama_adapter_lora_free(adapter)` -- release adapter
- Adapters are loaded against the model, activated against the context
- Multiple adapters can coexist in RAM; one active on the context at a time

**Key design decision:** LoRA adapters get their own three-state lifecycle
(COLD/WARM/HOT) that mirrors but is independent of the base model's
COLD/WARM/ACTIVE states. An adapter cannot be HOT unless the base model is
ACTIVE. Multiple adapters can be WARM (in RAM) simultaneously. The
orchestrator decides which adapter to activate based on tier config.

```
Adapter States:
  COLD ──load()──→ WARM ──activate()──→ HOT
    ↑                ↑                    │
    │                └──deactivate()──────┘
    └────────────unload()─────────────────┘
```

**What this version does NOT include:**
- Training/fine-tuning LoRA adapters (v2.1.0)
- Multiple simultaneous active adapters (llama.cpp limitation: one per context)
- Adapter-aware KV cache management (adapter swap invalidates KV cache --
  accepted cost at ~50-200ms, optimization deferred)

## AdapterState Enum

New enum in types, parallel to ModelState:

```cpp
/// @brief LoRA adapter lifecycle state.
/// @version 1.9.2
///
/// Mirrors ModelState but for LoRA adapters. An adapter must be HOT
/// for its weights to influence generation. Multiple adapters can be
/// WARM simultaneously (loaded in RAM), but only one can be HOT per
/// inference context.
enum class AdapterState : int {
    COLD = 0,  ///< Not loaded. No resources consumed.
    WARM = 1,  ///< Loaded in RAM via llama_adapter_lora_init(). Ready to activate.
    HOT  = 2   ///< Active on GPU context via llama_set_adapter_lora(). Influencing generation.
};
```

## AdapterManager Class

Central class for adapter lifecycle. Owned by `ModelOrchestrator`, operates
on `LlamaCppBackend` instances. This is NOT a three-layer hierarchy -- there
is one implementation (llama.cpp adapters), so it is a single concrete class
like `HookRegistry`.

```cpp
/// @file include/entropic/inference/adapter_manager.h
/// @brief AdapterManager -- LoRA adapter lifecycle and hot-swap.
///
/// @par Responsibilities:
/// - Load/unload LoRA adapters against a base model
/// - Activate/deactivate adapters on an inference context
/// - Track adapter states (COLD/WARM/HOT)
/// - Enforce single-HOT constraint per context
/// - Provide adapter metadata for routing
///
/// @par Thread safety:
/// - state queries are lock-free (std::atomic in AdapterEntry)
/// - load/unload/swap acquire adapter_mutex_
/// - swap operations coordinate with InferenceBackend transition_mutex_
///   via the orchestrator (AdapterManager does NOT lock the backend)
///
/// @par Ownership:
/// Owned by ModelOrchestrator. One AdapterManager per engine instance.
/// Adapters are keyed by (base_model_path, adapter_name) to support
/// future multi-model scenarios.

#pragma once

#include <entropic/types/config.h>
#include <entropic/inference/backend.h>

#include <atomic>
#include <mutex>
#include <string>
#include <unordered_map>
#include <vector>

// Forward declarations (llama.cpp types, not in public header)
struct llama_model;
struct llama_context;
struct llama_adapter_lora;

namespace entropic {

class AdapterManager {
public:
    /// @brief Load a LoRA adapter into RAM (COLD -> WARM).
    /// @param name Unique identifier for this adapter.
    /// @param adapter_path Path to the LoRA .gguf file.
    /// @param model llama_model* the adapter targets.
    /// @param scale LoRA scaling factor (default 1.0).
    /// @return true on success.
    ///
    /// The adapter is loaded via llama_adapter_lora_init(). It stays
    /// in RAM until explicitly unloaded or the base model is unloaded.
    /// Multiple adapters can be WARM simultaneously.
    bool load(const std::string& name,
              const std::filesystem::path& adapter_path,
              llama_model* model,
              float scale = 1.0f);

    /// @brief Unload adapter (any state -> COLD).
    /// @param name Adapter identifier.
    ///
    /// If the adapter is HOT, deactivates first. Frees RAM via
    /// llama_adapter_lora_free(). No-op if already COLD.
    void unload(const std::string& name);

    /// @brief Activate adapter on context (WARM -> HOT).
    /// @param name Adapter identifier.
    /// @param ctx llama_context* to activate on.
    /// @return true on success.
    ///
    /// If another adapter is currently HOT, deactivates it first
    /// (automatic swap). Fires ON_ADAPTER_SWAP hook via hook_interface_.
    bool activate(const std::string& name, llama_context* ctx);

    /// @brief Deactivate current HOT adapter (HOT -> WARM).
    /// @param ctx llama_context* to clear adapter from.
    ///
    /// Clears the LoRA adapter from the context. The adapter remains
    /// WARM in RAM. No-op if no adapter is HOT.
    void deactivate(llama_context* ctx);

    /// @brief Swap to a different adapter in one operation.
    /// @param name Target adapter identifier.
    /// @param ctx llama_context* to swap on.
    /// @return true on success.
    ///
    /// Equivalent to deactivate(ctx) + activate(name, ctx) but
    /// atomic from the orchestrator's perspective (holds adapter_mutex_
    /// for the entire operation). If the target adapter is COLD,
    /// returns false -- caller must load() first.
    bool swap(const std::string& name, llama_context* ctx);

    /// @brief Unload all adapters for a given base model.
    /// @param model The base model being unloaded.
    ///
    /// Called by orchestrator when base model transitions out of ACTIVE/WARM.
    /// Prevents dangling adapter handles.
    void unload_all_for_model(llama_model* model);

    // -- Queries (lock-free where possible) --

    /// @brief Get adapter state.
    AdapterState state(const std::string& name) const;

    /// @brief Get metadata for an adapter.
    /// @return AdapterInfo, or empty info with COLD state if not found.
    AdapterInfo info(const std::string& name) const;

    /// @brief List all known adapters.
    std::vector<AdapterInfo> list_adapters() const;

    /// @brief Get the currently HOT adapter name (empty if none).
    std::string active_adapter() const;

    /// @brief Set hook interface for ON_ADAPTER_SWAP dispatch.
    void set_hook_interface(const HookInterface* hooks);

private:
    /// @brief Internal entry for a managed adapter.
    struct AdapterEntry {
        std::string name;
        std::filesystem::path path;
        llama_adapter_lora* handle = nullptr;  ///< llama.cpp adapter handle
        llama_model* model = nullptr;          ///< Base model this targets
        float scale = 1.0f;
        std::atomic<AdapterState> state{AdapterState::COLD};
        size_t ram_bytes = 0;
        std::string tier_name;
        std::unordered_map<std::string, std::string> metadata;
    };

    /// @brief All managed adapters, keyed by name.
    std::unordered_map<std::string, AdapterEntry> adapters_;

    /// @brief Currently HOT adapter name (empty if none).
    std::string active_name_;

    /// @brief Guards all state transitions and the adapters_ map.
    mutable std::mutex adapter_mutex_;

    /// @brief Hook dispatch interface (optional, set by orchestrator).
    const HookInterface* hooks_ = nullptr;

    /// @brief Fire ON_ADAPTER_SWAP hook.
    /// @return true if hook allows the swap (not cancelled).
    bool fire_swap_hook(const std::string& current, const std::string& target,
                        const std::filesystem::path& target_path);
};

} // namespace entropic
```

boundary -- pure C, opaque handles.

    const char* adapter_path,
    const char* base_model_path,

/// @brief Unload a LoRA adapter.
/// @param handle Engine handle.

/// @return ENTROPIC_OK on success. Idempotent if already COLD.
///
/// @version 1.9.2
ENTROPIC_EXPORT entropic_error_t entropic_adapter_unload(
    entropic_handle_t handle,

## Orchestrator Integration

The `ModelOrchestrator` gains an `AdapterManager` member and coordinates
adapter swaps during tier transitions.

### Swap Sequence

```
User message arrives
    │
    ▼
Router classifies → target tier = "eng"
    │
    ▼
Orchestrator::get_model("eng")
    │
    ├── Base model already ACTIVE (same .gguf as previous tier)? → skip model swap
    │   │
    │   ▼
    │   Tier has adapter_path?
    │   ├── Yes → adapter_manager_.swap("eng", ctx_)
    │   │         ├── Deactivate current adapter (if any) → WARM
    │   │         ├── Fire ON_ADAPTER_SWAP hook (can cancel)
    │   │         ├── Activate "eng" adapter → HOT
    │   │         └── Log swap timing
    │   └── No  → adapter_manager_.deactivate(ctx_)
    │             (clear any active adapter — generate with base model only)
    │
    ├── Different base model needed → full model swap (existing v1.8.2 logic)
    │   │
    │   ▼
    │   After base model swap, load + activate tier's adapter (if configured)
    │
    ▼
Generate with adapted model
```

### Key Behaviors

1. **Adapter follows tier.** When the orchestrator routes to a tier, it
   ensures that tier's adapter is HOT (or no adapter if the tier has none).
   This is automatic -- the consumer does not manually manage adapters.

2. **Adapters pre-loaded at startup.** During `initialize()`, the orchestrator
   scans all tier configs for `adapter_path`. All referenced adapters are
   loaded to WARM at startup. This makes the first swap to any tier fast
   (~50-200ms activate only, no disk I/O).

3. **Adapter cleared on base model unload.** When a base model transitions
   from ACTIVE to WARM or COLD, `unload_all_for_model()` is called.
   Adapters targeting that model go COLD. They are reloaded when the base
   model becomes ACTIVE again.

4. **Deduped base model with different adapters.** The common case: all tiers
   share one base model, each with a different adapter (or no adapter). Tier
   swap = adapter swap only. Base model stays ACTIVE. This is the fast path.

5. **No adapter = base model only.** Tiers without `adapter_path` generate
   using the base model directly. If the previous tier had an adapter,
   it is deactivated (WARM) before generating.

## LlamaCppBackend Changes

`LlamaCppBackend` exposes its llama.cpp handles to `AdapterManager` via
protected accessors. `AdapterManager` is a friend class, or the orchestrator
passes the raw pointers. The handles are NOT exposed through the C API.

```cpp
/// @brief Added to LlamaCppBackend for v1.9.2.
/// @par These accessors are internal to librentropic-inference.so.
/// They are NOT part of the C API boundary.

/// @brief Get the loaded llama_model pointer.
/// @return nullptr if state is COLD.
llama_model* llama_model_ptr() { return model_; }

/// @brief Get the active llama_context pointer.
/// @return nullptr if state is not ACTIVE.
llama_context* llama_context_ptr() { return ctx_; }
```

The base `InferenceBackend` class is NOT modified. Adapter management is
layered on top via composition (AdapterManager), not inheritance. The backend
does not know about adapters -- the orchestrator coordinates both.

## KV Cache Invalidation

Swapping a LoRA adapter changes the model's effective weights. The existing
KV cache entries were computed with the previous adapter's weights and are
now stale.

**v1.9.2 strategy: clear KV cache on adapter swap.**

After `llama_set_adapter_lora()`, the orchestrator calls `llama_kv_cache_clear(ctx)`.
This forces re-evaluation of the prompt on the next generation. The cost
is re-processing the system prompt and conversation history tokens, typically
500-2000 tokens at ~10,000 tok/s = 50-200ms.

This is acceptable for v1.9.2. Future optimizations:
- v1.8.3 prompt caching: re-inject cached system prompt prefix after adapter swap
  (saves ~50% of re-evaluation cost for long system prompts)
- Selective KV cache invalidation (not supported by llama.cpp currently)

## Validation Criteria

### V1: Load adapter to WARM

Load a LoRA adapter via `entropic_adapter_load()`. Verify state transitions
from COLD to WARM. Verify `entropic_adapter_state()` returns 1 (WARM).
Verify `entropic_adapter_info()` returns correct path and scale.

### V2: Activate adapter to HOT

Load then activate a LoRA adapter. Verify state is HOT (2). Verify
`active_adapter()` returns the adapter name. Verify generation produces
different output than base model alone (use a known adapter that shifts
style).

### V3: Swap between adapters

Load two adapters (eng, qa). Activate eng. Swap to qa. Verify eng goes
WARM, qa goes HOT. Verify `active_adapter()` returns "qa". Verify the
swap is atomic (no window where both are HOT or neither is HOT).

### V4: Swap latency under 200ms

Measure wall-clock time for adapter swap (WARM adapter, already-loaded
base model). Swap latency includes: deactivate old, activate new, KV cache
clear. Target: < 200ms on reference hardware (16GB VRAM, NVMe SSD).

### V5: Tier transition triggers adapter swap

Configure two tiers with different adapters. Route to tier A (adapter A
goes HOT). Route to tier B (adapter A goes WARM, adapter B goes HOT).
Verify through RoutingResult that `adapter_name` and `adapter_swap_ms`
are populated correctly.

### V6: Tier without adapter clears active adapter

Configure tier A with adapter, tier B without. Route to A (adapter HOT).
Route to B (adapter deactivated, base model only). Verify
`active_adapter()` returns empty string.

### V7: ON_ADAPTER_SWAP hook fires and can cancel

Register an ON_ADAPTER_SWAP hook. Trigger adapter swap. Verify hook fires
with correct `current_adapter`, `target_adapter`, and `adapter_path`.
Register a cancelling hook (returns non-zero). Verify swap does NOT
proceed and adapter state is unchanged.

### V8: Base model unload cascades to adapters

Load a base model, load two adapters against it, activate one. Unload the
base model. Verify both adapters transition to COLD. Verify no dangling
llama_adapter_lora handles.

### V9: Config adapter_path validation

Set `adapter_path` in tier config to a valid file, an invalid file, and
a missing file. Verify: valid file loads successfully; invalid file returns
`ENTROPIC_ERROR_ADAPTER_LOAD_FAILED`; missing file logs WARNING at config
load time and returns `ENTROPIC_ERROR_LOAD_FAILED` at adapter load time.

### V10: Adapter preload at startup

Configure 3 tiers with adapter_path. Call `orchestrator.initialize()`.
Verify all 3 adapters are WARM before any generation occurs. Verify
startup log shows adapter load timing.

### V11: KV cache cleared after swap

Swap adapters, then generate. Verify the generation does not use stale
KV cache from the previous adapter. Measure the re-evaluation cost
(should be visible in generation timing as increased prompt processing).

### V12: Concurrent adapter state query during swap

From one thread, execute adapter swap. From another thread, query
`entropic_adapter_state()` repeatedly. Verify no data races
(ThreadSanitizer clean). Verify state transitions are atomic (never
observe an intermediate state).

### V13: Duplicate adapter name rejected

Attempt to load two different adapters with the same name. Verify the
second load returns `ENTROPIC_ERROR_INVALID_CONFIG`.

### V14: SLOC and complexity

All functions pass knots quality gates: cognitive complexity <= 15,
SLOC <= 50, returns <= 3.

### V15: Doxygen complete

Every public symbol in `adapter_manager.h` and updated type headers has
`@brief`, `@param`, `@return`, `@version`.

  WHEN unload("eng") is called
  THEN state("eng") returns COLD

  WHEN swap("qa", ctx) is called

  GIVEN tier config with 3 tiers, 2 having adapter_path

  WHEN orchestrator unloads model M (tier swap to different model)
