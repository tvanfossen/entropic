## Scope

Foundation for the entire C++ engine rewrite. After v1.8.0, the repo has:

- A CMake build system that compiles, links, and tests against llama.cpp
- All header-only dependencies vendored in `extern/`
- The `librentropic-types` library with error types, enums, and config structs
- A skeleton `librentropic` facade with lifecycle stubs in `entropic.h`
- `entropic_config.h` generated from CMake options
- `ENTROPIC_EXPORT` macro and symbol visibility enforced
- spdlog initialization pattern used by all subsequent code
- Catch2 BDD-style test framework with first tests running
- Doxygen configured and generating from heavy inline comments
- CI pipeline: GitHub Actions for CPU build + lint + static analysis
- Concrete base class pattern demonstrated and documented in code
- llama.cpp config pass-through fields defined in `ModelConfig`

This version produces NO runtime behavior. It is pure infrastructure. The
first "hello world" inference happens in v1.8.2.

### Phase 2: entropic_config.h.in Template

`include/entropic/entropic_config.h.in`:

```c
/**
 * @file entropic_config.h
 * @brief Generated build configuration — DO NOT EDIT.
 *
 * Generated by CMake from entropic_config.h.in. Source code reads
 * CONFIG_ENTROPIC_* defines from this file only. Never check CMake
 * variables directly in source. This makes the configuration frontend
 * swappable (CMake today, Kconfig later).
 */

#pragma once

/* ── Version ──────────────────────────────────────────── */
#define CONFIG_ENTROPIC_VERSION_MAJOR @entropic_VERSION_MAJOR@
#define CONFIG_ENTROPIC_VERSION_MINOR @entropic_VERSION_MINOR@
#define CONFIG_ENTROPIC_VERSION_PATCH @entropic_VERSION_PATCH@
#define CONFIG_ENTROPIC_VERSION_STRING "@entropic_VERSION@"

/* ── Build type ───────────────────────────────────────── */
#cmakedefine01 CONFIG_ENTROPIC_SHARED
#cmakedefine01 CONFIG_ENTROPIC_STATIC

/* ── Inference backend ────────────────────────────────── */
#cmakedefine01 CONFIG_ENTROPIC_CUDA
#cmakedefine01 CONFIG_ENTROPIC_VULKAN
#cmakedefine01 CONFIG_ENTROPIC_CPU_ONLY
#cmakedefine01 CONFIG_ENTROPIC_HAS_CUDA

/* ── MCP servers ──────────────────────────────────────── */
#cmakedefine01 CONFIG_ENTROPIC_MCP_FILESYSTEM
#cmakedefine01 CONFIG_ENTROPIC_MCP_BASH
#cmakedefine01 CONFIG_ENTROPIC_MCP_GIT
#cmakedefine01 CONFIG_ENTROPIC_MCP_DIAGNOSTICS
#cmakedefine01 CONFIG_ENTROPIC_MCP_WEB
#cmakedefine01 CONFIG_ENTROPIC_MCP_ENTROPIC

/* ── Storage ──────────────────────────────────────────── */
#cmakedefine01 CONFIG_ENTROPIC_STORAGE_SQLITE

/* ── Features ─────────────────────────────────────────── */
#cmakedefine01 CONFIG_ENTROPIC_STREAMING

/* ── Paths ────────────────────────────────────────────── */
/** @brief Compile-time path to bundled data files (prompts, grammars, models).
 *  Overridable at runtime via config_dir field in config YAML. */
#define CONFIG_ENTROPIC_DATA_DIR "@ENTROPIC_DATA_DIR@"
```

CMake generates `CONFIG_ENTROPIC_SHARED` as `1` or `0` (not defined/undefined)
via `#cmakedefine01`. This avoids the fragile `#ifdef` vs `#if` ambiguity.
Source code uses `#if CONFIG_ENTROPIC_CUDA` consistently.

The CMake variables feeding the template require a naming bridge:

### Phase 3: ENTROPIC_EXPORT Macro

`include/entropic/entropic_export.h`:

```c
/**
 * @file entropic_export.h
 * @brief Symbol visibility macro for all exported symbols.
 *
 * Applied ONLY to:
 * - Factory functions (entropic_create_server, entropic_create_inference_backend)
 * - C API functions in entropic.h
 * - entropic_plugin_api_version()
 *
 * All other symbols are hidden by default (CMAKE_CXX_VISIBILITY_PRESET=hidden).
 */

#pragma once

#if defined(ENTROPIC_STATIC_BUILD)
    /* Static build — no import/export decoration */
    #define ENTROPIC_EXPORT
#elif defined(_WIN32)
    #if defined(ENTROPIC_BUILDING_DLL)
        #define ENTROPIC_EXPORT __declspec(dllexport)
    #else
        #define ENTROPIC_EXPORT __declspec(dllimport)
    #endif
#else
    #define ENTROPIC_EXPORT __attribute__((visibility("default")))
#endif
```

The `ENTROPIC_BUILDING_DLL` define is set per-target in CMake via
`target_compile_definitions(entropic-types PRIVATE ENTROPIC_BUILDING_DLL)`.
Consumers linking the `.so` do NOT define it, so they get `dllimport` on
Windows and default visibility on Linux (where import is a no-op).

For static builds (`ENTROPIC_STATIC=ON`), the top-level CMakeLists sets
`ENTROPIC_STATIC_BUILD` globally, collapsing the macro to nothing.

### Phase 4: Error Types (librentropic-types)

`include/entropic/types/error.h`:

```c
/**
 * @file error.h
 * @brief Error types for cross-.so error reporting.
 *
 * Exceptions do NOT cross .so boundaries. All cross-boundary error
 * reporting uses:
 * 1. Return codes — C API functions return entropic_error_t.
 * 2. Error callbacks — consumer registers for async errors.
 * 3. Error state on handles — entropic_last_error() returns last message.
 *
 * @par Thread safety
 * entropic_last_error() is per-handle, not global. Each handle maintains
 * its own error state. Thread-safe for different handles; concurrent
 * access to the SAME handle requires external synchronization.
 */

#pragma once

#include <stddef.h>
#include <entropic/entropic_export.h>

#ifdef __cplusplus
extern "C" {
#endif

/**
 * @brief Error codes returned by all C API functions.
 *
 * ENTROPIC_OK (0) on success. All other values indicate failure.
 * Use entropic_error_name() to get a human-readable string for logging.
 */
typedef enum {
    ENTROPIC_OK = 0,                          ///< Success
    ENTROPIC_ERROR_INVALID_ARGUMENT,          ///< NULL pointer, empty string, out-of-range value
    ENTROPIC_ERROR_INVALID_CONFIG,            ///< Config validation failed (missing fields, bad values)
    ENTROPIC_ERROR_INVALID_STATE,             ///< Operation not valid in current state (e.g., generate before activate)
    ENTROPIC_ERROR_MODEL_NOT_FOUND,           ///< Model file does not exist at resolved path
    ENTROPIC_ERROR_LOAD_FAILED,               ///< Model load failed (corrupt file, OOM, unsupported format)
    ENTROPIC_ERROR_GENERATE_FAILED,           ///< Generation failed (context overflow, model error)
    ENTROPIC_ERROR_TOOL_NOT_FOUND,            ///< Requested tool not registered
    ENTROPIC_ERROR_PERMISSION_DENIED,         ///< Tool call blocked by permission manager
    ENTROPIC_ERROR_PLUGIN_VERSION_MISMATCH,   ///< Plugin API version != engine expected version
    ENTROPIC_ERROR_PLUGIN_LOAD_FAILED,        ///< dlopen/dlsym failed on plugin .so
    ENTROPIC_ERROR_TIMEOUT,                   ///< Operation exceeded time limit
    ENTROPIC_ERROR_CANCELLED,                 ///< Operation cancelled via cancel token
    ENTROPIC_ERROR_OUT_OF_MEMORY,             ///< Allocation failed (system RAM or VRAM)
    ENTROPIC_ERROR_IO,                        ///< File/network I/O error
    ENTROPIC_ERROR_INTERNAL,                  ///< Unexpected internal error (bug)
} entropic_error_t;

/**
 * @brief Opaque handle to an entropic engine instance.
 *
 * All per-handle functions (including entropic_last_error) operate on this.
 * Created by entropic_create(), destroyed by entropic_destroy().
 */
typedef struct entropic_engine* entropic_handle_t;

/**
 * @brief Get the last error message for a handle.
 *
 * Returns the error message from the most recent failed operation on this
 * handle. The returned string is owned by the handle and valid until the
 * next API call on the same handle.
 *
 * @param handle Engine handle (may be NULL for global errors during create).
 * @return Null-terminated error message string, or "" if no error.
 *         Owned by the handle — do NOT free.
 */
ENTROPIC_EXPORT const char* entropic_last_error(entropic_handle_t handle);

/**
 * @brief Get the human-readable name for an error code.
 *
 * @param code Error code.
 * @return Static string like "ENTROPIC_ERROR_LOAD_FAILED". Never NULL.
 *
 * @par Example
 * @code
 * entropic_error_t err = entropic_configure(h, cfg);
 * if (err != ENTROPIC_OK) {
 *     spdlog::error("configure failed: {} — {}",
 *         entropic_error_name(err), entropic_last_error(h));
 * }
 * @endcode
 */
ENTROPIC_EXPORT const char* entropic_error_name(entropic_error_t code);

/**
 * @brief Error callback type for async error reporting.
 *
 * Registered via entropic_set_error_callback(). Called from the thread
 * where the error occurs. The callback MUST NOT call back into the
 * entropic API (deadlock risk). Keep callbacks short — log and return.
 *
 * @param code Error code.
 * @param message Null-terminated error message. Valid only for callback duration.
 * @param user_data Opaque pointer passed during registration.
 */
typedef void (*entropic_error_callback_t)(
    entropic_error_t code,
    const char* message,
    void* user_data);

/**
 * @brief Register an error callback on a handle.
 *
 * Only one callback per handle. Setting a new one replaces the previous.
 * Pass NULL to remove.
 *
 * @param handle Engine handle.
 * @param callback Callback function, or NULL to remove.
 * @param user_data Opaque pointer forwarded to callback.
 * @return ENTROPIC_OK on success, ENTROPIC_ERROR_INVALID_ARGUMENT if handle is NULL.
 */
ENTROPIC_EXPORT entropic_error_t entropic_set_error_callback(
    entropic_handle_t handle,
    entropic_error_callback_t callback,
    void* user_data);

#ifdef __cplusplus
}
#endif
```

### Phase 5: entropic.h — Public C API Interface Sketch

`include/entropic/entropic.h`:

```c
/**
 * @file entropic.h
 * @brief Public C API for the Entropic inference engine.
 *
 * This is the unified facade. Most consumers link against librentropic
 * and include only this header. All functions are pure C — no C++ types
 * cross this boundary.
 *
 * @par Lifecycle
 * @code
 *   entropic_handle_t h = NULL;
 *   entropic_create(&h);
 *   entropic_configure(h, config_json);
 *   entropic_run(h, "Hello");             // blocking single-turn
 *   entropic_destroy(h);
 * @endcode
 *
 * @par Thread safety
 * A single handle is NOT thread-safe. Use one handle per thread, or
 * synchronize externally. Different handles are fully independent.
 *
 * @par Error handling
 * All functions return entropic_error_t. On failure, call
 * entropic_last_error(handle) for the detailed message.
 *
 * @par Memory ownership
 * Functions returning char* transfer ownership to the caller.
 * Free with entropic_free(). Consumers providing buffers the engine
 * will free (e.g., hook modified_json) MUST allocate with
 * entropic_alloc(). Strings returned as const char* are owned by
 * the handle and valid until the next call on that handle.
 */

#pragma once

#include <entropic/entropic_config.h>
#include <entropic/entropic_export.h>
#include <entropic/types/error.h>

#ifdef __cplusplus
extern "C" {
#endif

/* ── Lifecycle ────────────────────────────────────────── */

/**
 * @brief Create a new engine instance.
 *
 * Allocates and initializes an engine handle. The engine is idle until
 * entropic_configure() is called.
 *
 * @param[out] handle Pointer to receive the new handle.
 * @return ENTROPIC_OK on success, ENTROPIC_ERROR_OUT_OF_MEMORY on failure.
 */
ENTROPIC_EXPORT entropic_error_t entropic_create(entropic_handle_t* handle);

/**
 * @brief Configure the engine from a JSON config string.
 *
 * Parses the JSON, validates against the config schema, resolves model
 * paths, and prepares the engine for operation. Does NOT load models
 * (that happens on first generate or explicit activate).
 *
 * JSON is used at the C API boundary because it's universal — any language
 * can produce it. Config FILES are YAML (parsed by ryml inside
 * librentropic-config.so). Use entropic_configure_from_file() to load
 * YAML config files directly without manual JSON conversion.
 *
 * @param handle Engine handle.
 * @param config_json JSON string with engine configuration.
 * @return ENTROPIC_OK, ENTROPIC_ERROR_INVALID_CONFIG, ENTROPIC_ERROR_INVALID_ARGUMENT.
 */
ENTROPIC_EXPORT entropic_error_t entropic_configure(
    entropic_handle_t handle,
    const char* config_json);

/**
 * @brief Configure engine from a YAML or JSON config file.
 *
 * Convenience wrapper that reads the file, parses it (YAML via ryml or
 * JSON via nlohmann/json based on extension), and applies the configuration.
 * Internally calls entropic_configure() after conversion.
 *
 * Most consumers use this — game engines, embedded systems, and CLI tools
 * all load config from files rather than constructing JSON programmatically.
 *
 * @param handle Engine handle.
 * @param config_path Path to YAML (.yaml/.yml) or JSON (.json) config file.
 * @return ENTROPIC_OK, ENTROPIC_ERROR_INVALID_CONFIG, ENTROPIC_ERROR_INVALID_ARGUMENT.
 */
ENTROPIC_EXPORT entropic_error_t entropic_configure_from_file(
    entropic_handle_t handle,
    const char* config_path);

/**
 * @brief Destroy an engine instance and free all resources.
 *
 * Unloads models, closes storage, destroys all child objects.
 * After this call, the handle is invalid. Passing NULL is a no-op.
 *
 * @param handle Engine handle to destroy.
 */
ENTROPIC_EXPORT void entropic_destroy(entropic_handle_t handle);

/* ── Version ──────────────────────────────────────────── */

/**
 * @brief Get the library version string.
 * @return Static string like "1.8.0". Never NULL. Do NOT free.
 */
ENTROPIC_EXPORT const char* entropic_version(void);

/**
 * @brief Get the plugin API version number.
 *
 * Plugins call this to verify compatibility before registering.
 *
 * @return Integer API version (incremented on breaking changes).
 */
ENTROPIC_EXPORT int entropic_api_version(void);

/* ── Memory ───────────────────────────────────────────── */

/**
 * @brief Free a string allocated by the engine.
 *
 * Allocate a buffer using the engine's allocator. All cross-boundary
 * allocations (hook modified_json, consumer-provided strings that the
 * engine will free) MUST use this function. Paired with entropic_free().
 *
 * @param size Number of bytes to allocate.
 * @return Pointer to allocated memory, or NULL on failure.
 * @version 1.8.0
 */
ENTROPIC_EXPORT void* entropic_alloc(size_t size);

/**
 * All char* return values that transfer ownership MUST be freed with
 * this function. All buffers allocated with entropic_alloc() MUST be
 * freed with this function. Passing NULL is a no-op.
 *
 * @param ptr Pointer to free (from engine return value or entropic_alloc).
 * @version 1.8.0
 */
ENTROPIC_EXPORT void entropic_free(void* ptr);

/* ── Generation (stubs — implemented in v1.8.4) ──────── */

/**
 * @brief Single-turn blocking generation.
 *
 * @param handle Engine handle.
 * @param input User message (null-terminated).
 * @param[out] result_json JSON result string. Caller must free with entropic_free().
 * @return ENTROPIC_OK, ENTROPIC_ERROR_GENERATE_FAILED, ENTROPIC_ERROR_INVALID_STATE.
 *
 * @note Stub in v1.8.0. Returns ENTROPIC_ERROR_INVALID_STATE until v1.8.4.
 */
ENTROPIC_EXPORT entropic_error_t entropic_run(
    entropic_handle_t handle,
    const char* input,
    char** result_json);

/**
 * @brief Streaming generation with token callback.
 *
 * @param handle Engine handle.
 * @param input User message.
 * @param on_token Called for each generated token. Must be thread-safe.
 * @param user_data Forwarded to on_token.
 * @param cancel_flag Pointer to int. Set to non-zero to cancel. May be NULL.
 * @return ENTROPIC_OK, ENTROPIC_ERROR_GENERATE_FAILED, ENTROPIC_ERROR_CANCELLED.
 *
 * @note Stub in v1.8.0. Returns ENTROPIC_ERROR_INVALID_STATE until v1.8.4.
 */
```

### Phase 8: spdlog Initialization Pattern

`include/entropic/types/logging.h`:

```cpp
/**
 * @file logging.h
 * @brief spdlog initialization and logger access.
 *
 * Every .so in the entropic project uses spdlog for structured logging.
 * This header establishes the initialization pattern used from v1.8.0
 * onward. All loggers are created through this interface.
 *
 * @par Pattern
 * Each library creates a named logger via entropic::log::get("libname").

 * Libraries call get() lazily — sink config propagates automatically via
 * spdlog's registry.

 * spdlog loggers are thread-safe. Initialization (init_logging) must

#pragma once

#include <spdlog/spdlog.h>
#include <spdlog/sinks/stdout_color_sinks.h>
#include <memory>
#include <string>

namespace entropic::log {

/**

 *
 * Call once at engine startup (in entropic_create). Sets the global
 * log level and output format. Subsequent calls are no-ops.
 *
 * @param level spdlog level (trace, debug, info, warn, error, critical).
 */
void init(spdlog::level::level_enum level = spdlog::level::info);

/**
 * @brief Get or create a named logger.
 *
 * Returns the existing logger if already created, or creates a new one
 * sharing the root sink configuration.
 *
 * @param name Logger name (e.g., "types", "inference", "mcp").
 * @return Shared pointer to the logger.
 *
 * @par Example
 * @code
 * auto log = entropic::log::get("inference");
 * log->info("Model loaded: {} ({:.1f} GB)", path, size_gb);
 * log->error("Load failed: {}", entropic_error_name(err));
 * @endcode
 */
std::shared_ptr<spdlog::logger> get(const std::string& name);

} // namespace entropic::log
```

Implementation in `src/types/logging.cpp`:

```cpp
#include <entropic/types/logging.h>
#include <spdlog/sinks/stdout_color_sinks.h>
#include <mutex>

namespace entropic::log {

static std::once_flag s_init_flag;
static std::shared_ptr<spdlog::sinks::stdout_color_sink_mt> s_sink;

void init(spdlog::level::level_enum level) {
    std::call_once(s_init_flag, [level]() {
        s_sink = std::make_shared<spdlog::sinks::stdout_color_sink_mt>();
        s_sink->set_level(level);
        spdlog::set_default_logger(
            std::make_shared<spdlog::logger>("entropic", s_sink));
        spdlog::set_level(level);
        spdlog::set_pattern("[%Y-%m-%d %H:%M:%S.%e] [%n] [%^%l%$] %v");
    });
}

std::shared_ptr<spdlog::logger> get(const std::string& name) {
    auto logger = spdlog::get(name);
    if (logger) {
        return logger;
    }
    // Create with shared sink. If init() hasn't been called yet,
    // create a default sink (defensive — init should be called first).
    if (!s_sink) {
        init(spdlog::level::info);
    }
    logger = std::make_shared<spdlog::logger>(name, s_sink);
    spdlog::register_logger(logger);
    return logger;
}

} // namespace entropic::log
```

### Phase 9: Concrete Base Class Pattern

The three-layer hierarchy is the core C++ design pattern. Here is the
reference implementation established in v1.8.0, using `InferenceBackend`
as the canonical example. Other subsystems (MCP, storage, config) follow
the same structure.

```
Layer 1: C interface (i_inference_backend.h)
  - Pure C, opaque handles, function pointers
  - This IS the .so boundary
  - Defined in v1.8.0 as reserved stubs

Layer 2: Concrete base class (backend.h)
  - C++ class inside the .so
  - Contains 80%+ of logic: lifecycle state machine, locking, metrics,
    logging, error conversion, streaming dispatch
  - Public methods are non-virtual (Template Method pattern)
  - Protected pure virtual do_*() methods for subclass specifics

Layer 3: Implementation (e.g., LlamaCppBackend)
  - 20% override: only the do_*() methods
  - No lifecycle logic, no locking, no metrics
  - Pure domain-specific code
```

Example demonstrating the pattern (not shipped as production code —
the real InferenceBackend ships in v1.8.2):

```cpp
// ── Layer 2: Concrete base class ────────────────────────
// include/entropic/inference/backend.h (internal to inference .so)

namespace entropic {

class InferenceBackend {
public:
    /// @brief Load model from config. Transitions COLD → WARM.
    /// Base class handles: state validation, locking, logging, metrics.
    bool load(const ModelConfig& config) {
        std::lock_guard lock(transition_mutex_);
        auto expected = ModelState::COLD;
        if (!state_.compare_exchange_strong(expected, ModelState::WARM)) {
            log_->error("load() called in state {}, expected COLD",
                        state_name(expected));
            return false;
        }
        log_->info("Loading model: {}", config.path.string());
        auto start = std::chrono::steady_clock::now();

        if (!do_load(config)) {
            state_.store(ModelState::COLD);
            log_->error("do_load() failed for {}", config.path.string());
            return false;
        }

        auto elapsed = std::chrono::steady_clock::now() - start;
        log_->info("Model loaded in {:.1f}s",
            std::chrono::duration<double>(elapsed).count());
        return true;
    }

    /// @brief Lock-free state query. Safe from any thread.
    ModelState state() const { return state_.load(); }

protected:
    /// @brief Subclass implements actual model loading.
    virtual bool do_load(const ModelConfig& config) = 0;

private:
    std::atomic<ModelState> state_{ModelState::COLD};
    std::mutex transition_mutex_;
    std::shared_ptr<spdlog::logger> log_ = log::get("inference");
};

// ── Layer 3: Thin implementation ────────────────────────
// src/inference/llama_cpp_backend.h (internal to inference .so)

class LlamaCppBackend : public InferenceBackend {
protected:
    /// @brief 20%: Only the llama.cpp-specific model load call.
    bool do_load(const ModelConfig& config) override {
        llama_model_params params = llama_model_default_params();
        params.n_gpu_layers = config.gpu_layers;
        params.use_mlock = config.use_mlock;
        model_ = llama_model_load_from_file(
            config.path.c_str(), params);
        return model_ != nullptr;
    }

private:
    llama_model* model_ = nullptr;
};
```

The key invariant: `load()` is NOT virtual. Subclasses cannot skip the
state machine, locking, or logging. They can ONLY override `do_load()`.
This enforces the 80/20 split at the language level.

   Zero undocumented symbol warnings for files in `include/entropic/`.

   GitHub Actions build (gcc, Debug/Release) all green.
   Static analysis (clang-tidy + cppcheck + knots) clean.

8. **llama.cpp submodule checks out:**
   ```bash
   git submodule update --init --recursive

   ```
   Submodule present at pinned commit.

   Every `entropic_error_t` value has a name via `entropic_error_name()`.

    Multiple calls to `entropic::log::init()` are idempotent.
    `entropic::log::get()` returns consistent loggers.

## Dependencies

- **v1.7.3** — Engine API cleanup must be complete so the Python engine's
  public API boundary is stabilized before the C++ port begins. The C API
  in `entropic.h` mirrors the cleaned-up Python API surface.
- **CMake >= 3.21** — Required for `cmake --preset` support.
- **C++20 compiler** — gcc (latest). clang optional for clang-tidy linting.
  Needed for `std::filesystem`, concepts (future use), `std::format` (future use).
- **Make** — Default generator (Unix Makefiles). Consumers can override with
  `-G Ninja` or any other CMake generator
  but all presets use it.
