[SYSTEM DIRECTIVE: SELF-DIAGNOSIS]

Analyze your recent actions and identify any issues. Follow these steps:

1. Call entropic.diagnose to get a full engine state snapshot.
2. Review the tool call history for:
   - Repeated failures (same tool, same error)
   - Duplicate tool calls (circuit breaker risk)
   - Tool calls that returned errors
   - Unexpected state (wrong phase, wrong tier)
3. Review your reasoning for:
   - Actions that didn't achieve the stated goal
   - Unnecessary tool calls
   - Missing context that led to errors
4. Produce a structured assessment:
   - FINDINGS: What went wrong (be specific)
   - ROOT CAUSE: Why it went wrong (reasoning error vs engine issue vs config issue)
   - RECOMMENDATION: What to do differently

Be honest and specific. The goal is accurate self-assessment, not self-defense.
