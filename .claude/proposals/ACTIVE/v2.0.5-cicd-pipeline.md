## Scope

End-to-end automation: PR checks, nightly builds, release pipeline,
artifact publishing. Replaces the current manual `inv build && inv test`
workflow with enforced gating.

---

## Workflows

### 1. PR Check (`ci.yml`)

Triggers on every PR to `develop` and `main`.

| Step | What | Time |
|------|------|------|
| Pre-commit | All hooks (knots, flake8, doxygen-guard, ruff) | ~30s |
| Build (CPU) | `inv build --cpu` | ~2min |
| Unit tests | `inv test --cpu --no-build` (673+ tests) | ~5s |
| C99 compat | Header compilation test | included |
| Coverage | lcov/gcov → report artifact | ~10s |
| Wrapper gen | Verify auto-generated wrapper matches committed | ~5s |

**Gate:** All steps must pass for merge.

### 2. Nightly (`nightly.yml`)

Triggers on schedule (02:00 UTC) against `develop`.

| Step | What |
|------|------|
| Build (CUDA) | `inv build --full` with GPU runner |
| Unit tests (GPU) | Full test suite on CUDA build |
| Model tests | `inv test-model` against primary model |
| Benchmark | `inv benchmark` with regression detection |
| Coverage | Full coverage report → badge update |

**Output:** `results.json` committed to `benchmark/results/` on pass.
Failures open a GitHub Issue tagged `nightly-failure`.

### 3. Release (`release.yml`)

Triggers on version tag push (`v*`).

| Step | What |
|------|------|
| Validate tag | Tag matches `CMakeLists.txt` `project(VERSION ...)` |
| Build matrix | CPU + CUDA for Linux x86_64 |
| Package .so | Versioned shared libraries + headers |
| Python wheel | Build `entropic-engine` wheel from wrapper |
| Changelog | Generate from conventional commits since last tag |
| GitHub Release | Create release with artifacts + changelog |
| PyPI publish | Upload wheel to PyPI |

## Test Gating

| Gate | When | What |
|------|------|------|
| PR merge | Every PR | Unit tests (CPU, fast) |
| Nightly | Daily | Full suite + model tests + benchmarks |
| Minor release (x.y.0) | Tag push | Full model test suite must have passed in recent nightly |
| Patch release (x.y.z) | Tag push | Unit tests only (CUDA build) |

### Coverage

- Tool: `lcov` + `gcov` (GCC) or `llvm-cov` (Clang)
- Gate: No coverage regression >2% on PR (warning, not blocking)
- Badge: Updated nightly → README
- Reports: Uploaded as workflow artifact

### Benchmark Regression

- Compare against `benchmark/results/baseline.json`
- Alert on >10% throughput regression (tok/s)
- Alert on >15% latency regression (TTFT)
- Nightly updates baseline on pass

## Secrets & Infrastructure

| Secret | Purpose | Where |
|--------|---------|-------|
| `PYPI_TOKEN` | Publish entropic-engine wheel | Release workflow |
| `GH_TOKEN` | Create releases, comment on PRs | Built-in |
| GPU runner | CUDA builds, model tests, benchmarks | Self-hosted |

### Self-Hosted GPU Runner

- Label: `self-hosted, gpu, cuda`
- Hardware: NVIDIA GPU with >= 16GB VRAM
- Software: CUDA 12.x toolkit, CMake 3.25+, GCC 13
- Registration: `actions-runner` on the GPU machine
- Security: Runs only on trusted branches (develop, main, release tags)

## Validation Criteria

1. PR to develop triggers CI → all hooks + build + tests pass
2. Nightly runs unattended → model tests + benchmarks complete
3. `git tag v2.0.5 && git push --tags` triggers release workflow
4. GitHub Release created with .so tarball + wheel attached
5. `pip install entropic-engine` installs the published wheel
6. Coverage report accessible as workflow artifact
7. Benchmark regression detected when intentionally degraded
