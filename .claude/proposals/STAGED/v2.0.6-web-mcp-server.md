## HTTP Client: cpp-httplib

**Library:** [yhirose/cpp-httplib](https://github.com/yhirose/cpp-httplib)
- Header-only (single header, ~25K lines)
- MIT license (compatible with LGPL-3.0)
- SSL via OpenSSL (already available on build machines)
- No additional dependencies beyond OpenSSL

**Alternative considered:** libcurl — heavier, not header-only, more
complex CMake integration. cpp-httplib is sufficient for synchronous
fetch operations.

### Integration

```cmake
# extern/CMakeLists.txt
FetchContent_Declare(
    cpp-httplib
    GIT_REPOSITORY https://github.com/yhirose/cpp-httplib
    GIT_TAG        v0.18.1
)
FetchContent_MakeAvailable(cpp-httplib)
```

Link: `target_link_libraries(entropic-mcp PRIVATE httplib::httplib)`

## Tool Specifications

### `web.web_fetch`

Fetches a URL and returns the text content.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `url` | string | (required) | URL to fetch (http/https only) |
| `max_length` | int | 20000 | Max response body chars (capped at 50000) |
| `headers` | object | `{}` | Additional HTTP headers |
| `timeout_ms` | int | 10000 | Request timeout in milliseconds |

**Behavior:**
1. Validate URL scheme (http/https only — no file://, ftp://, etc.)
2. Set User-Agent: `entropic/{version}`
3. Follow redirects (max 5)
4. Fetch response body
5. Strip HTML tags if Content-Type is text/html (basic tag stripping)
6. Truncate to `max_length` characters
7. Return text content + metadata (status code, content-type, final URL)

**Response format:**
```json
{
  "result": "Page content here...",
  "metadata": {
    "status_code": 200,
    "content_type": "text/html",
    "final_url": "https://example.com/page",
    "content_length": 15234,
    "truncated": false
  }
}
```

**Error cases:**
- Invalid URL scheme → tool error
- Connection failed → tool error with reason
- Timeout → tool error
- Non-2xx status → return body with status in metadata (let model decide)

### `web.web_search`

Executes a web search and returns results.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `query` | string | (required) | Search query |
| `max_results` | int | 5 | Results to return (capped at 10) |

**Search backend options (in priority order):**

1. **SearXNG** (preferred) — self-hosted, no API key, JSON API
   - Config: `mcp.web.searxng_url: "http://localhost:8080"`
   - Endpoint: `GET /search?q={query}&format=json&categories=general`

2. **Brave Search API** — free tier (1 req/sec, 2000/month)
   - Config: `mcp.web.brave_api_key: "BSA..."`
   - Endpoint: `GET https://api.search.brave.com/res/v1/web/search`

3. **DuckDuckGo HTML** — no API key, parse HTML results
   - Config: none needed (fallback)
   - Endpoint: `GET https://html.duckduckgo.com/html/?q={query}`

**Response format:**
```json
{
  "result": "1. Title - URL\n   Snippet text\n\n2. ...",
  "metadata": {
    "backend": "searxng",
    "result_count": 5,
    "query": "original query"
  }
}
```

## Configuration

```yaml
mcp:
  enable_web: true        # Default: false
  web:
    searxng_url: ""       # SearXNG instance URL (empty = disabled)
    brave_api_key: ""     # Brave Search API key (empty = disabled)
    user_agent: ""        # Custom User-Agent (empty = default)
    timeout_ms: 10000     # Default request timeout
    max_redirects: 5      # Max redirect follows
```

## Security

- **Off by default** — must be explicitly enabled
- **URL scheme validation** — only http:// and https://
- **No local network access** — reject private/loopback IPs (10.x, 172.16-31.x, 192.168.x, 127.x, ::1)
- **Rate limiting** — max 1 request per second per tool (configurable)
- **Timeout enforcement** — hard 30s cap regardless of config
- **No cookie/session persistence** — each request is stateless
- **Content size cap** — 1MB raw download limit before truncation
- **Permission system** — governed by standard MCP permissions (auto_approve or callback)
