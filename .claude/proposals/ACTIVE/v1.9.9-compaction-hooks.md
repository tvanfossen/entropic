## Scope

Makes the compaction system programmatically extensible. After v1.9.9,
consumers can invoke compaction on demand per identity context, register
custom compactors that replace or wrap the default, and receive pre/post
compaction callbacks with full context.

This builds on:
- **v1.8.4** (Engine Loop): `CompactionManager` with deterministic
  value-density compaction, `ContextManager` that triggers compaction at
  threshold, `TokenCounter`
- **v1.9.1** (Hook Architecture): `ENTROPIC_HOOK_ON_PRE_COMPACT` (hook
  point 11) and `ENTROPIC_HOOK_ON_POST_COMPACT` (hook point 12) already
  defined with pre-modify/cancel and post-transform semantics

**Current behavior (v1.8.4):**
- Compaction triggers automatically when `ContextManager::check_compaction()`
  detects token usage above threshold
- Single compaction strategy: value-density structured extraction (no model)
- Compaction is internal -- no consumer control over when or how it happens
- `CompactionManager::check_and_compact()` returns `CompactionResult` with
  token counts and summary
- PRE_COMPACT/POST_COMPACT hooks (v1.9.1) fire but cannot replace the
  compaction algorithm itself

**What v1.9.9 adds:**
- `ICompactor` interface: consumer-replaceable compaction function
- Per-identity compactor registration (different identities, different strategies)
- Programmatic compaction trigger via C API (`entropic_compact()`)
- Default compactor wraps the existing v1.8.4 value-density strategy
- Consumer compactor can use model inference for summarization, custom logic,
  or wrap the default with additional processing
- Compaction context includes identity name, enabling identity-aware strategies
- PRE_COMPACT hook context extended with `identity` field

**What v1.9.9 does NOT include:**
- Model-based summarization built-in (consumer provides if desired)
- Compaction quality metrics or benchmarks
- Multi-pass compaction (single compactor call per trigger)

## Architecture

### Compaction Call Chain

```
Trigger: threshold reached OR entropic_compact() called
    |
    v
ContextManager::check_compaction(ctx, force)
    |
    v
fire_pre(ON_PRE_COMPACT, context_json)
    |
    +-- cancelled? --> return unchanged
    |
    v
CompactorRegistry::compact(identity, messages, config)
    |
    +-- per-identity compactor registered? --> call it
    |
    +-- no --> call default compactor (v1.8.4 value-density)
    |
    v
fire_post(ON_POST_COMPACT, result_json)
    |
    +-- post-hook transforms? --> apply
    |
    v
Return compacted messages + CompactionResult
```

### Compactor Function Signature

The compactor is a C function pointer at the `.so` boundary. Internally,
the C++ side wraps it in a more ergonomic interface.

**C boundary (in `entropic.h`):**

```c
/**
 * @brief Compactor function type.
 *
 * A compactor takes a JSON array of messages, a configuration JSON,
 * and produces a compacted JSON array of messages plus a summary string.
 *
 * @param messages_json   JSON array of messages to compact.
 * @param config_json     Compaction configuration (threshold, identity, etc.).
 * @param out_messages    Output: compacted messages JSON. Caller frees with entropic_free().
 * @param out_summary     Output: summary text. Caller frees with entropic_free(). May be NULL.
 * @return 0 on success, non-zero on failure.
 *
 * @par Config JSON schema:
 * @code
 * {
 *   "identity": "eng",
 *   "token_count": 12000,
 *   "max_tokens": 16384,
 *   "threshold_percent": 0.75,
 *   "force": false
 * }
 * @endcode
 *
 * @version 1.9.9
 */
typedef int (*entropic_compactor_fn)(
    const char* messages_json,
    const char* config_json,
    char** out_messages,
    char** out_summary,
    void* user_data);
```

The `user_data` pointer lets consumers carry state (model handle, custom
config, etc.) without global variables.

### Why a Function Pointer, Not a Hook

PRE_COMPACT hooks can cancel or modify the input. POST_COMPACT hooks can
transform the output. But neither REPLACES the compaction algorithm itself.

A custom compactor is a fundamentally different extension point: it IS the
compaction, not an observer of it. The hook system is wrong for this because:

1. Hooks chain -- multiple hooks modify sequentially. Compaction needs
   exactly ONE strategy per invocation, not a chain of partial strategies.
2. Hooks have priority ordering. Compactor selection is identity-based
   dispatch, not priority-based composition.
3. The default compactor is not a hook -- it is the engine's built-in
   behavior. Custom compactors REPLACE it, not augment it.

The relationship is:

```
PRE_COMPACT hooks  -->  Compactor (default OR custom)  -->  POST_COMPACT hooks
   (modify input)        (THE compaction logic)              (transform output)
```

Hooks and compactors compose but do not overlap.

## CompactorRegistry

Manages per-identity compactor registration. Lives in `librentropic-core.so`.
Single concrete class (no three-layer hierarchy -- one implementation, same
pattern as `HookRegistry`).

```cpp
/// @file include/entropic/core/compactor_registry.h
/// @brief Per-identity compactor registration and dispatch.
///
/// @par Thread safety:
/// Registration/deregistration take a write lock. Dispatch takes a
/// read lock + snapshot. Safe to register from one thread while
/// compacting on another.
///
/// @par Ownership:
/// Owned by the engine handle. One CompactorRegistry per engine instance.
/// The default compactor (value-density) is registered at construction.

#pragma once

#include <entropic/types/config.h>
#include <functional>
#include <mutex>
#include <shared_mutex>
#include <string>
#include <unordered_map>
#include <vector>

namespace entropic {

/// @brief Internal C++ compactor function type.
/// @version 1.9.9
///
/// Takes messages + config, returns compacted messages + summary.
/// Wraps the C function pointer with C++ types for internal use.
using CompactorFn = std::function<CompactionResult(
    const std::vector<Message>& messages,
    const CompactionConfig& config,
    const std::string& identity)>;

/// @brief A registered compactor entry.
/// @version 1.9.9
struct CompactorEntry {
    entropic_compactor_fn c_callback;  ///< C function pointer (NULL for C++ compactors)
    void* user_data;                   ///< Opaque data for C callback
    CompactorFn cpp_fn;                ///< C++ function (wraps c_callback or native)
};

/// @brief Per-identity compactor registry and dispatch.
/// @version 1.9.9
class CompactorRegistry {
public:
    /// @brief Construct with default compactor.
    /// @param default_compaction_manager Reference to v1.8.4 CompactionManager.
    explicit CompactorRegistry(CompactionManager& default_manager);

    /// @brief Register a compactor for a specific identity.
    /// @param identity Identity name ("" for global fallback).
    /// @param compactor C function pointer.
    /// @param user_data Opaque pointer passed to compactor.
    /// @return ENTROPIC_OK on success.
    ///
    /// Replaces any previously registered compactor for this identity.
    /// Pass identity="" to replace the global default.
    entropic_error_t register_compactor(
        const std::string& identity,
        entropic_compactor_fn compactor,
        void* user_data);

    /// @brief Deregister a compactor for a specific identity.
    /// @param identity Identity name ("" for global fallback).
    /// @return ENTROPIC_OK on success (idempotent).
    ///
    /// After deregistration, the identity falls back to the global
    /// compactor, then to the built-in default.
    entropic_error_t deregister_compactor(const std::string& identity);

    /// @brief Run compaction using the appropriate compactor.
    /// @param identity Current identity name.
    /// @param messages Messages to compact.
    /// @param config Compaction configuration.
    /// @return CompactionResult with compacted messages and metadata.
    ///
    /// Resolution order:
    /// 1. Per-identity compactor (if registered)
    /// 2. Global custom compactor (identity="", if registered)
    /// 3. Built-in default (v1.8.4 value-density)
    CompactionResult compact(
        const std::string& identity,
        const std::vector<Message>& messages,
        const CompactionConfig& config);

    /// @brief Check if a custom compactor is registered for an identity.
    /// @param identity Identity name.
    /// @return true if a non-default compactor will be used.
    bool has_custom_compactor(const std::string& identity) const;

private:
    /// @brief Wrap a C function pointer into a CompactorFn.
    /// @param compactor C function pointer.
    /// @param user_data Opaque pointer.
    /// @return C++ callable wrapping the C callback.
    CompactorFn wrap_c_compactor(
        entropic_compactor_fn compactor,
        void* user_data) const;

    /// Per-identity compactors. Key "" is the global fallback.
    std::unordered_map<std::string, CompactorEntry> compactors_;

    /// Built-in default compactor (v1.8.4 CompactionManager).
    CompactorFn default_compactor_;

    mutable std::shared_mutex mutex_;  ///< Guards compactors_ map
};

} // namespace entropic
```

### Resolution Logic

```
compact(identity, messages, config):
    1. Read-lock mutex_
    2. IF compactors_[identity] exists: use it
    3. ELSE IF compactors_[""] exists: use it (global custom)
    4. ELSE: use default_compactor_ (v1.8.4 value-density)
    5. Unlock
    6. Call selected compactor
    7. Return result
```

### Default Compactor Wrapper

The default compactor wraps `CompactionManager::check_and_compact()` from
v1.8.4. It strips the threshold check (that is done by `ContextManager`
before calling `CompactorRegistry::compact()`), and directly runs the
value-density strategy:

```cpp
/// @brief Default compactor using v1.8.4 value-density strategy.
/// @version 1.9.9
CompactorFn make_default_compactor(CompactionManager& manager);
```

This is a free function that returns a lambda capturing the
`CompactionManager` reference. The lambda calls the internal `_compact()`
method (which needs to be exposed as a public method renamed to
`compact_messages()` for clarity).

## Extended CompactionResult

v1.8.4's `CompactionResult` is extended with identity and compactor metadata:

```cpp
/// @brief Result of a compaction operation.
/// @version 1.9.9 — extended from v1.8.4
struct CompactionResult {
    bool compacted;                      ///< Whether compaction occurred
    int old_token_count;                 ///< Tokens before compaction
    int new_token_count;                 ///< Tokens after compaction
    std::string summary;                 ///< Structured summary text (may be empty)
    int preserved_messages;              ///< Messages kept after compaction
    int messages_summarized;             ///< Messages absorbed into summary
    std::vector<Message> messages;       ///< The compacted message list

    /* v1.9.9 additions */
    std::string identity;                ///< Identity that triggered compaction
    std::string compactor_source;        ///< "default", "global_custom", or identity name
    bool custom_compactor_used;          ///< true if a consumer-provided compactor ran
};
```

The `messages` field is new -- v1.8.4 returned messages via out-parameter.
Including them in the result makes the compactor function signature cleaner
(one return value instead of tuple).

## C API Additions

### Programmatic Compaction Trigger

```c
/**
 * @brief Trigger compaction on the current context.
 *
 * Forces compaction regardless of threshold. Useful for consumers that
 * want to compact before a known-large operation (e.g., before delegation,
 * before loading a large file, before switching identity).
 *
 * The compaction runs through the full pipeline:
 * PRE_COMPACT hooks -> compactor (default or custom) -> POST_COMPACT hooks.
 *
 * @param handle        Engine handle.
 * @param identity      Identity context for compactor selection (NULL = current).
 * @param result_json   Output: compaction result JSON. Caller frees with entropic_free().
 *                      NULL if no compaction occurred.
 *
 * @par Result JSON schema:
 * @code
 * {
 *   "compacted": true,
 *   "old_token_count": 12000,
 *   "new_token_count": 3500,
 *   "preserved_messages": 8,
 *   "messages_summarized": 37,
 *   "identity": "eng",
 *   "compactor_source": "default",
 *   "custom_compactor_used": false
 * }
 * @endcode
 *
 * @return ENTROPIC_OK on success.
 *         - ENTROPIC_ERROR_INVALID_HANDLE
 *         - ENTROPIC_ERROR_COMPACTION_FAILED (compactor returned error)
 *
 * @threadsafety Serialized per-handle.
 * @version 1.9.9
 */
ENTROPIC_EXPORT entropic_error_t entropic_compact(
    entropic_handle_t handle,
    const char* identity,
    char** result_json);
```

### Compactor Registration

```c
/**
 * @brief Register a custom compactor for an identity.
 *
 * The compactor replaces the default compaction strategy for the
 * specified identity. Pass identity="" to set a global fallback
 * that applies to all identities without a per-identity compactor.
 *
 * Resolution order when compaction triggers:
 * 1. Per-identity compactor (if registered)
 * 2. Global custom compactor (identity="", if registered)
 * 3. Built-in default (value-density structured extraction)
 *
 * Only one compactor per identity. Re-registering replaces the previous.
 *
 * @param handle     Engine handle.
 * @param identity   Identity name, or "" for global fallback.
 * @param compactor  Compactor function pointer.
 * @param user_data  Opaque pointer passed to compactor on each call.
 * @return ENTROPIC_OK on success.
 *         - ENTROPIC_ERROR_INVALID_HANDLE
 *         - ENTROPIC_ERROR_INVALID_CONFIG (NULL compactor)
 *
 * @threadsafety Serialized per-handle.
 * @version 1.9.9
 */
ENTROPIC_EXPORT entropic_error_t entropic_register_compactor(
    entropic_handle_t handle,
    const char* identity,
    entropic_compactor_fn compactor,
    void* user_data);

/**
 * @brief Deregister a custom compactor for an identity.
 *
 * After deregistration, the identity falls back to the global custom
 * compactor (if any), then to the built-in default.
 *
 * @param handle     Engine handle.
 * @param identity   Identity name, or "" for global fallback.
 * @return ENTROPIC_OK on success (idempotent).
 *         - ENTROPIC_ERROR_INVALID_HANDLE
 *
 * @threadsafety Serialized per-handle.
 * @version 1.9.9
 */
ENTROPIC_EXPORT entropic_error_t entropic_deregister_compactor(
    entropic_handle_t handle,
    const char* identity);
```

## Extended Hook Context

### ON_PRE_COMPACT (v1.9.9 extension)

v1.9.1 defined the PRE_COMPACT context as:

```json
{"messages": [...], "token_count": 12000, "threshold": 10000, "force": false}
```

v1.9.9 extends it with identity and compactor info:

```json
{
  "messages": [...],
  "token_count": 12000,
  "threshold": 10000,
  "force": false,
  "identity": "eng",
  "compactor": "default",
  "custom_compactor_registered": false
}
```

The `compactor` field is `"default"`, `"global_custom"`, or the identity
name (matching `CompactionResult::compactor_source`). Pre-hooks can use
this to make decisions based on which compactor will run.

### ON_POST_COMPACT (v1.9.9 extension)

v1.9.1 defined:

```json
{"original_count": 45, "compacted_count": 12, "tokens_before": 12000, "tokens_after": 3500}
```

v1.9.9 extends:

```json
{
  "original_count": 45,
  "compacted_count": 12,
  "tokens_before": 12000,
  "tokens_after": 3500,
  "identity": "eng",
  "compactor_source": "default",
  "custom_compactor_used": false,
  "summary": "Original task: ..."
}
```

## Integration with ContextManager

`ContextManager::check_compaction()` (v1.8.4) currently calls
`CompactionManager::check_and_compact()` directly. v1.9.9 changes this
to route through `CompactorRegistry`:

```
ContextManager::check_compaction(ctx, force):
    1. Count tokens (existing)
    2. Check threshold (existing)
    3. fire_pre(ON_PRE_COMPACT, context_json)     -- existing from v1.9.1
       - cancelled? return unchanged
       - modified? use modified messages/config
    4. compactor_registry_.compact(                -- NEW: was CompactionManager direct call
           ctx.locked_tier,                        -- identity name for dispatch
           ctx.messages,
           compaction_config_)
    5. fire_post(ON_POST_COMPACT, result_json)     -- existing from v1.9.1
       - transformed? apply
    6. Apply result to ctx.messages
    7. Return
```

The change is surgical: replace one method call (step 4) with a call to
`CompactorRegistry` which internally dispatches to the default or custom
compactor.

## Consumer Use Cases

### Use Case 1: Model-Based Summarization

A TUI consumer that uses the model itself to generate richer summaries:

```c
int model_compactor(
    const char* messages_json,
    const char* config_json,
    char** out_messages,
    char** out_summary,
    void* user_data)
{
    // user_data carries inference handle
    // 1. Parse messages
    // 2. Build summarization prompt from messages
    // 3. Call inference to generate summary
    // 4. Assemble: system + summary_msg + recent user messages
    // 5. Serialize to out_messages, out_summary
    return 0;
}

entropic_register_compactor(handle, "", model_compactor, inference_handle);
```

### Use Case 2: Per-Identity Strategy

A game with different compaction needs per NPC type:

```c
// Dialog NPCs: aggressive compaction, keep only last 5 exchanges
entropic_register_compactor(handle, "npc_dialog", aggressive_compactor, NULL);

// Quest NPCs: preserve all quest-related messages
entropic_register_compactor(handle, "npc_quest", quest_compactor, quest_state);

// All others: default value-density strategy (no registration needed)
```

### Use Case 3: Wrapping the Default

A consumer that adds audit logging around the default compaction:

```c
int auditing_compactor(
    const char* messages_json,
    const char* config_json,
    char** out_messages,
    char** out_summary,
    void* user_data)
{
    AuditContext* ctx = (AuditContext*)user_data;

    // Log pre-compaction state
    audit_log(ctx, "pre_compact", messages_json);

    // Call the default compactor
    int rc = ctx->default_compactor(
        messages_json, config_json, out_messages, out_summary,
        ctx->default_user_data);

    // Log post-compaction state
    audit_log(ctx, "post_compact", *out_messages);

    return rc;
}
```

For this use case, the consumer needs access to the default compactor as
a callable. The C API provides this:

```c
/**
 * @brief Get the built-in default compactor function pointer.
 *
 * Useful for consumers that want to wrap the default behavior
 * (e.g., add logging, metrics, or post-processing around it).
 *
 * @param compactor  Output: default compactor function pointer.
 * @param user_data  Output: user_data to pass to the default compactor.
 * @return ENTROPIC_OK on success.
 *
 * @version 1.9.9
 */
ENTROPIC_EXPORT entropic_error_t entropic_get_default_compactor(
    entropic_compactor_fn* compactor,
    void** user_data);
```

### Use Case 4: Programmatic Compaction Before Delegation

Before delegating to a child identity, compact the parent's context to
free up token budget:

```c
// About to delegate a large task
char* result = NULL;
entropic_compact(handle, "lead", &result);
// Now delegate with more headroom
entropic_delegate(handle, "eng", task_json);
entropic_free(result);
```

## CompactionManager Changes (v1.8.4 Modification)

The v1.8.4 `CompactionManager` needs one change: expose the internal
`_compact()` method as a public API for the default compactor wrapper.

```cpp
/// @brief Compact messages using value-density strategy.
/// @param messages Messages to compact.
/// @return CompactionResult with compacted messages and metadata.
///
/// Public in v1.9.9 (was private _compact in v1.8.4).
/// Used by the default compactor wrapper in CompactorRegistry.
///
/// @version 1.9.9
CompactionResult compact_messages(const std::vector<Message>& messages);
```

This is a rename and visibility change, not a logic change. The method
body is identical to v1.8.4's `_compact()`.

## Validation Criteria

### V1: Default compaction unchanged

Without any custom compactor registration, compaction behavior is
identical to v1.8.4. Same token reduction, same summary format, same
message preservation. Regression test against existing compaction tests.

### V2: Programmatic compaction via C API

Call `entropic_compact(handle, "eng", &result)`. Verify compaction runs
(regardless of threshold). Verify result JSON contains expected fields.
Verify context messages are compacted.

### V3: Custom compactor replaces default

Register a custom compactor that returns a single summary message.
Trigger compaction. Verify the default strategy does NOT run. Verify
the custom compactor's output is used.

### V4: Per-identity compactor dispatch

Register compactor A for identity "eng", compactor B for "qa". Trigger
compaction for "eng". Verify compactor A runs. Trigger for "qa". Verify
compactor B runs. Trigger for "lead" (no registration). Verify default
runs.

### V5: Global custom compactor fallback

Register a global compactor (identity=""). Trigger compaction for "eng"
(no per-identity registration). Verify the global compactor runs.
Register per-identity for "eng". Trigger again. Verify per-identity
takes precedence over global.

### V6: Deregistration falls back

Register custom compactor for "eng". Verify it runs. Deregister it.
Trigger compaction for "eng". Verify default compactor runs.

### V7: PRE_COMPACT hook sees extended context

Register a PRE_COMPACT hook. Trigger compaction. Verify context_json
contains `identity`, `compactor`, `custom_compactor_registered` fields.

### V8: POST_COMPACT hook sees extended context

Register a POST_COMPACT hook. Trigger compaction. Verify context_json
contains `identity`, `compactor_source`, `custom_compactor_used`, `summary`.

### V9: PRE_COMPACT cancellation prevents compaction

Register a PRE_COMPACT hook returning non-zero. Trigger compaction.
Verify no compactor runs. Verify messages unchanged.

### V10: POST_COMPACT transformation applied

Register a POST_COMPACT hook that modifies `tokens_after` in the result.
Verify the engine uses the transformed result.

### V11: Custom compactor failure is non-fatal

Register a custom compactor that returns non-zero. Trigger compaction.
Verify a WARNING is logged. Verify the engine falls back to the default
compactor (not crash, not skip compaction entirely).

### V12: Get default compactor for wrapping

Call `entropic_get_default_compactor(&fn, &data)`. Verify the returned
function pointer is non-NULL. Call it directly with test messages.
Verify it produces valid compacted output matching the default strategy.

### V13: Thread safety

Register compactors from one thread while triggering compaction from
another. Verify no data races (ThreadSanitizer clean).

### V14: Compaction during active loop

Trigger `entropic_compact()` from a PRE_GENERATE hook (mid-loop).
Verify compaction runs, context shrinks, and the loop continues with
the compacted context.

### V15: SLOC and complexity

All functions pass knots quality gates: cognitive complexity <= 15,
SLOC <= 50, returns <= 3.

### V16: Doxygen complete

Every public symbol in `compactor_registry.h` and updated `entropic.h`

## Unit Tests

### test_compactor_registry.cpp

```
SCENARIO: Default compactor runs when no custom registered
  GIVEN a CompactorRegistry with default CompactionManager
  AND no custom compactors registered
  WHEN compact("eng", messages, config) is called
  THEN the default value-density strategy runs
  AND result.custom_compactor_used == false
  AND result.compactor_source == "default"

SCENARIO: Per-identity compactor overrides default
  GIVEN a custom compactor registered for "eng"
  WHEN compact("eng", messages, config) is called
  THEN the custom compactor runs (not default)
  AND result.custom_compactor_used == true
  AND result.compactor_source == "eng"

SCENARIO: Global custom compactor applies to unregistered identities
  GIVEN a custom compactor registered for "" (global)
  AND no per-identity compactor for "qa"
  WHEN compact("qa", messages, config) is called
  THEN the global custom compactor runs
  AND result.compactor_source == "global_custom"

SCENARIO: Per-identity takes precedence over global
  GIVEN a global custom compactor
  AND a per-identity compactor for "eng"
  WHEN compact("eng", messages, config) is called
  THEN the per-identity compactor runs (not global)

SCENARIO: Deregistration falls back to global
  GIVEN a per-identity compactor for "eng"
  AND a global custom compactor
  WHEN deregister_compactor("eng") is called
  AND compact("eng", messages, config) is called
  THEN the global custom compactor runs

SCENARIO: Deregistration falls back to default
  GIVEN a per-identity compactor for "eng"
  AND no global custom compactor
  WHEN deregister_compactor("eng") is called
  AND compact("eng", messages, config) is called
  THEN the default compactor runs

SCENARIO: Re-registration replaces previous
  GIVEN compactor A registered for "eng"
  WHEN compactor B is registered for "eng"
  AND compact("eng", messages, config) is called
  THEN compactor B runs (not A)

SCENARIO: Custom compactor failure falls back to default
  GIVEN a custom compactor for "eng" that returns non-zero
  WHEN compact("eng", messages, config) is called
  THEN a WARNING is logged
  AND the default compactor runs as fallback
  AND result.compactor_source == "default"

SCENARIO: Custom compactor receives correct config JSON
  GIVEN a custom compactor that captures its config_json
  WHEN compact("eng", messages, config) is called
  THEN config_json contains "identity": "eng"
  AND config_json contains "token_count" and "max_tokens"

SCENARIO: has_custom_compactor returns correct state
  GIVEN no custom compactors
  WHEN has_custom_compactor("eng") is called
  THEN it returns false
  WHEN register_compactor("eng", ...) is called
  THEN has_custom_compactor("eng") returns true
  WHEN deregister_compactor("eng") is called
  THEN has_custom_compactor("eng") returns false

SCENARIO: Concurrent registration and dispatch
  GIVEN a CompactorRegistry
  WHEN thread A registers compactors while thread B calls compact()
  THEN no data races occur (ThreadSanitizer clean)

SCENARIO: NULL compactor rejected
  GIVEN a CompactorRegistry
  WHEN register_compactor("eng", NULL, NULL) is called
  THEN it returns ENTROPIC_ERROR_INVALID_CONFIG
```

### test_compaction_hooks.cpp

```
SCENARIO: Programmatic compaction via entropic_compact
  GIVEN an engine with conversation context at 80% capacity
  WHEN entropic_compact(handle, "eng", &result) is called
  THEN compaction runs (force=true)
  AND result_json contains compacted metrics
  AND context messages are reduced

SCENARIO: entropic_compact fires PRE and POST hooks
  GIVEN PRE_COMPACT and POST_COMPACT hooks registered
  WHEN entropic_compact(handle, NULL, &result) is called
  THEN PRE_COMPACT fires before compaction
  AND POST_COMPACT fires after compaction
  AND hook context_json includes identity field

SCENARIO: PRE_COMPACT cancellation prevents entropic_compact
  GIVEN a PRE_COMPACT hook returning non-zero
  WHEN entropic_compact(handle, "eng", &result) is called
  THEN result_json is NULL
  AND no compactor runs
  AND messages are unchanged

SCENARIO: Custom compactor integrates with automatic threshold
  GIVEN a custom compactor for "eng"
  AND context exceeds compaction threshold
  WHEN the engine loop calls check_compaction()
  THEN the custom compactor runs (not default)
  AND POST_COMPACT hook context shows custom_compactor_used: true

SCENARIO: entropic_get_default_compactor returns callable
  GIVEN an engine handle
  WHEN entropic_get_default_compactor(&fn, &data) is called
  THEN fn is non-NULL
  AND calling fn with test messages produces valid compacted output

SCENARIO: Wrapping compactor calls default internally
  GIVEN a wrapping compactor that calls the default and adds logging
  WHEN compaction triggers
  THEN the wrapper runs
  AND the default strategy produces the actual compacted messages
  AND the wrapper's logging runs around it

SCENARIO: Identity passed through from LoopContext
  GIVEN a custom compactor for "eng" that records the identity it receives
  AND the engine loop routed to tier "eng"
  WHEN automatic compaction triggers
  THEN the compactor receives identity "eng"

SCENARIO: Compaction with no active identity uses empty string
  GIVEN a global custom compactor
  WHEN entropic_compact(handle, NULL, &result) is called
  AND no tier is locked in the current context
  THEN the compactor receives identity ""
  AND the global compactor runs
```
