## Scope

The engine logs almost nothing about its own work. A consumer watching
logs sees tokens go in and silence. No generation results, no routing
decisions, no timing, no throughput, no grammar enforcement, no profile
application, no compaction triggers, no tool execution flow, no
authorization decisions.

Exposed by v1.10.2 model tests: per-test log files captured only 3-4
engine-emitted lines per generation. Tests had to add their own logging
(`log_result()`, `spdlog::info("[testN]")`) just to prove anything
worked. That's backwards.

**This is not an 8-file patch. A full audit of all 71 source files found
30+ files that do meaningful work but don't log it.** The proposal
covers every file in the engine.

---

## Design Principles

1. **Every subsystem that does work logs what it did at INFO.** A
   consumer reading engine logs understands exactly what happened
   without reading source code.

2. **NEVER truncate log content.** No `substr(0, N)`, no `first N
   chars`, no summarizing. Full content, always. If a log is too
   noisy, move it to a different level — don't gut it.

3. **Default loud, opt-in quiet.** INFO is the workhorse. Consumers
   who want silence turn it down. The engine doesn't hide its work
   behind DEBUG by default.

4. **Logging is a consumer deliverable.** Engine logs are the primary
   debugging interface for anyone building on entropic — NPC designers,
   TUI authors, assistant builders. They need to see the engine's full
   thought process to understand how to adjust/tune the model.

5. **Pretty-print as infrastructure.** Formatting helpers live in the
   logging layer, not reinvented per call site: content formatting,
   timing, token stats, decision chains.

6. **Logprob is opt-in.** `evaluate_logprobs` has inference-time cost.
   The feature is off by default, enabled by consumer flag or test
   config. When on, results logged at INFO.

7. **Content escaping is mandatory.** All user-supplied or model-generated
   content MUST be escaped before passing to spdlog. Embedded newlines,
   ANSI escape sequences, and fmt format specifiers (`{}`, `{:x}`) in
   raw content will corrupt log structure or crash the formatter. The
   `log_content()` utility handles this — never pass raw content directly
   to `logger->info("{}", raw_content)`.

8. **Authorization detail at DEBUG.** ALLOW/DENY decisions log at INFO
   (identity, tool, result). Glob pattern matching detail, grant chains,
   and permission topology log at DEBUG. Consumers doing security testing
   can enable DEBUG; production consumers shouldn't see the full
   permission model by default.

## Known Infrastructure Issues (Must Fix First)

### 1. `now()` / `elapsed_ms()` duplicated across 3 files

Identical anonymous-namespace timing helpers exist in:
- `src/inference/orchestrator.cpp:32-45`
- `src/inference/backend.cpp:46-56`
- `src/inference/adapter_manager.cpp:33-43`

This proposal adds timing to 30+ more files. These MUST move to the
logging utilities BEFORE any call-site work, or every new file will
duplicate them again.

### 2. `entropic::log::get()` creates loggers with stale sinks

`logging.cpp:48` creates new loggers using `s_sink` — the original
console sink from init. The test infrastructure's `set_all_sinks()` at
`model_tests.cpp:456` uses `spdlog::apply_all()` which only reaches
loggers already in the registry. Any logger first created AFTER
`start_test_log()` runs gets the original console sink, not the test
file sink.

This means lazily-created loggers during test execution won't appear
in test log files. Fix: either `get()` checks for current global sinks,
or the test infrastructure re-applies sinks when new loggers register.
This MUST be fixed or the entire proposal's validation (reading per-test
log files) is unreliable.

### 3. Bare spdlog calls bypass named logger infrastructure

6 files use `spdlog::info/warn/error` directly instead of named loggers.
These go through the default "entropic" logger, making them unsearchable
by subsystem and invisible to test sinks after `set_all_sinks()`:

- `src/core/constitutional_validator.cpp` — `spdlog::warn()` x3
- `src/storage/backend.cpp` — `spdlog::info()` x3
- `src/storage/database.cpp` — `spdlog::info/error` x5
- `src/storage/permission_persister.cpp` — `spdlog::info/error`
- `src/storage/logger.cpp` — `spdlog::info/error`
- `src/storage/c_interface.cpp` — `spdlog::error` x5

## Infrastructure: Logging Utilities

Before touching individual files, add shared formatting helpers to the
logging layer (extend `types/logging.h` or new `types/log_format.h`):

| Utility | Purpose |
|---|---|
| `log_content(logger, level, label, text)` | Full text with content escaping (newlines, ANSI, fmt specifiers safely escaped) |
| `log_timing(logger, label, ms)` | Consistent `{label}: {ms:.1f}ms` |
| `log_tokens(logger, count, time_ms)` | `{count} tokens, {ms:.1f}ms, {tok_s:.1f} tok/s` |
| `log_decision(logger, label, key, value)` | `{label}: {key} -> {value}` |
| `elapsed_ms(start, end)` | Centralized timing — replaces 3 duplicated copies |
| `now()` | `std::chrono::steady_clock::now()` wrapper — replaces 3 duplicated copies |

These are used by all subsystems below. Call sites become one-liners.
All utilities MUST be implemented and tested before any call-site work.
