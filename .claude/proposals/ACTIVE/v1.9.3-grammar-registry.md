## Scope

Centralized grammar management for the inference engine. After v1.9.3, grammars
are named resources loadable by key, registerable at runtime, and overridable
per-generation call. This decouples grammar resolution from identity frontmatter
and enables cross-model grammar chaining (model A's output defines model B's
grammar constraint).

This builds on:
- v1.8.1 `ConfigLoader` -- bundled data directory discovery (`ENTROPIC_DATA_DIR`)
- v1.8.2 `InferenceBackend` / `LlamaCppBackend` -- `GenerationParams` struct,
  grammar string passed to `llama_sampler_init_grammar()`
- v1.8.4 `ResponseGenerator` -- system prompt assembly, per-tier generation kwargs
- v1.9.0 Auto-generated Python wrapper -- grammar C API surfaces in Python

**Current Python behavior (behavioral spec):**
- Identity frontmatter declares `grammar: filename.gbnf`
- `ModelOrchestrator._resolve_grammar()` resolves to file content string
- `_load_grammar_file()` caches content by resolved path
- Content string passed to `LlamaGrammar.from_string()` at generation time
- Three bundled grammars: `compactor.gbnf`, `scribe.gbnf`, `benchmark_judge.gbnf`

**What v1.9.3 changes:**
- Grammars are named resources in a registry, not ad-hoc path resolutions
- Runtime registration of new grammars without restart
- `GenerationParams` gains `grammar_key` (lookup by name) alongside raw `grammar` string
- In-memory GBNF strings registered without file backing
- Validation API: parse GBNF without generating, return structured errors
- Cross-model chaining: model A output → parse → register as grammar → constrain model B

**What v1.9.3 does NOT include:**
- Grammar composition (combining grammars at runtime) -- future work
- Grammar versioning (registry is flat key → content) -- unnecessary complexity
- GBNF compilation caching at the llama.cpp level -- llama.cpp handles this internally

## GrammarEntry Struct

Registry entries carry metadata alongside the GBNF content:

```cpp
/// @file include/entropic/types/config.h (addition)
/// @brief Metadata for a registered grammar.
/// @version 1.9.3
struct GrammarEntry {
    std::string key;            ///< Unique registry key (e.g., "compactor", "chess_executor")
    std::string gbnf_content;   ///< Raw GBNF grammar string
    std::string source;         ///< Origin: "bundled", "file", "runtime", "dynamic"
    bool validated = false;     ///< true if grammar has passed validation
    std::string error;          ///< Non-empty if validation failed
};
```

Source values:
- `"bundled"` -- loaded from `data/grammars/` at startup
- `"file"` -- loaded from a user-specified file path at runtime
- `"runtime"` -- registered programmatically via C API with in-memory GBNF string
- `"dynamic"` -- generated from another model's output (cross-model chaining)

## GrammarRegistry Class

Single concrete class (no three-layer hierarchy -- one implementation, like
`AdapterManager` and `HookRegistry`). Owned by `ModelOrchestrator`.

```cpp
/// @file include/entropic/inference/grammar_registry.h
/// @brief GrammarRegistry -- named grammar management and validation.
///
/// @par Responsibilities:
/// - Load bundled grammars from data/grammars/ at startup
/// - Register/deregister grammars by name at runtime
/// - Resolve grammar_key to GBNF content string for generation
/// - Validate GBNF grammar syntax without generating
///
/// @par Thread safety:
/// - get() and has() are lock-free (read from snapshot)
/// - register/deregister acquire registry_mutex_
/// - Validation is stateless and thread-safe
///
/// @par Ownership:
/// Owned by ModelOrchestrator. One GrammarRegistry per engine instance.
/// Replaces the _grammar_cache dict and _resolve_grammar() method from
/// the Python orchestrator.

#pragma once

#include <entropic/types/config.h>

#include <mutex>
#include <string>
#include <unordered_map>
#include <vector>
#include <filesystem>

namespace entropic {

class GrammarRegistry {
public:
    /// @brief Load all bundled grammars from a directory.
    /// @param grammar_dir Path to data/grammars/ directory.
    /// @return Number of grammars loaded.
    ///
    /// Scans for *.gbnf files. Each file's stem becomes the key
    /// (e.g., "compactor.gbnf" -> key "compactor"). Files are read,
    /// validated, and registered. Invalid files are logged as WARNING
    /// and skipped (not fatal).
    size_t load_bundled(const std::filesystem::path& grammar_dir);

    /// @brief Register a grammar by key with GBNF content string.
    /// @param key Unique grammar name.
    /// @param gbnf_content Raw GBNF grammar string.
    /// @param source Origin tag ("runtime", "dynamic", "file").
    /// @return true on success. false if key already exists (use deregister first).
    ///
    /// The grammar is validated on registration. If validation fails,
    /// the grammar is still registered (consumers may want to inspect
    /// the error) but entry.validated is false and entry.error is set.
    bool register_grammar(const std::string& key,
                          const std::string& gbnf_content,
                          const std::string& source = "runtime");

    /// @brief Register a grammar from a file path.
    /// @param key Unique grammar name (if empty, uses filename stem).
    /// @param path Path to .gbnf file.
    /// @return true on success. false if file unreadable or key exists.
    bool register_from_file(const std::string& key,
                            const std::filesystem::path& path);

    /// @brief Remove a grammar from the registry.
    /// @param key Grammar name to remove.
    /// @return true if removed. false if key not found.
    ///
    /// Bundled grammars can be deregistered (allows overriding defaults).
    bool deregister(const std::string& key);

    /// @brief Get GBNF content string for a grammar key.
    /// @param key Grammar name.
    /// @return GBNF content string, or empty string if not found.
    ///
    /// Returns content regardless of validation status. The caller
    /// (inference backend) will get a generation error if the grammar
    /// is malformed -- this is intentional (fail at generation, not
    /// at registration, to allow fixing and re-registering).
    std::string get(const std::string& key) const;

    /// @brief Check if a grammar key exists in the registry.
    /// @param key Grammar name.
    /// @return true if registered.
    bool has(const std::string& key) const;

    /// @brief Get full entry metadata for a grammar key.
    /// @param key Grammar name.
    /// @return GrammarEntry, or entry with empty key if not found.
    GrammarEntry entry(const std::string& key) const;

    /// @brief List all registered grammar keys.
    /// @return Vector of GrammarEntry metadata (without content, for efficiency).
    std::vector<GrammarEntry> list() const;

    /// @brief Validate a GBNF grammar string.
    /// @param gbnf_content Raw GBNF string to validate.
    /// @return Empty string on success, error description on failure.
    ///
    /// Parses the grammar using llama_grammar_init() with a throwaway
    /// sampler. Does not require a loaded model. This is a static
    /// utility -- can be called without registering the grammar.
    static std::string validate(const std::string& gbnf_content);

    /// @brief Number of registered grammars.
    size_t size() const;

    /// @brief Remove all registered grammars.
    void clear();

private:
    /// @brief All registered grammars, keyed by name.
    std::unordered_map<std::string, GrammarEntry> grammars_;

    /// @brief Guards mutations to grammars_ map.
    mutable std::mutex registry_mutex_;
};

} // namespace entropic
```

## GenerationParams Changes

The `GenerationParams` struct (v1.8.2) gains a `grammar_key` field:

```cpp
/// @brief Generation parameters passed to InferenceBackend::generate().
/// @version 1.9.3 -- added grammar_key
struct GenerationParams {
    // ... existing fields (temperature, top_p, top_k, max_tokens, stop, etc.) ...

    /// @brief Raw GBNF grammar string to constrain output.
    /// Takes precedence over grammar_key if both are set.
    /// Existing field from v1.8.2.
    std::string grammar;

    /// @brief Grammar registry key. Resolved to GBNF content by orchestrator
    /// before passing to the backend. If both grammar and grammar_key are set,
    /// grammar (raw string) takes precedence.
    /// @version 1.9.3
    std::string grammar_key;
};
```

**Resolution order (in orchestrator, before calling backend):**
1. `params.grammar` (raw GBNF string) -- highest priority, explicit override
2. `params.grammar_key` -- looked up in `GrammarRegistry`
3. Identity frontmatter `grammar:` field -- resolved to registry key (migration path)
4. None -- unconstrained generation

This preserves backward compatibility: existing code passing raw grammar strings
continues to work. The `grammar_key` field is additive.

## C API Additions

New functions in `entropic.h` for grammar management:

```c
/// @brief Register a grammar by key from a GBNF content string.
/// @param handle Engine handle.
/// @param key Unique grammar name.
/// @param gbnf_content Raw GBNF grammar string (null-terminated).
/// @return ENTROPIC_OK on success.
///         - ENTROPIC_ERROR_INVALID_CONFIG if key already exists
///
/// @version 1.9.3
ENTROPIC_EXPORT entropic_error_t entropic_grammar_register(
    entropic_handle_t handle,
    const char* key,
    const char* gbnf_content);

/// @brief Register a grammar from a file path.
/// @param handle Engine handle.
/// @param key Grammar name (if NULL, uses filename stem).
/// @param path Path to .gbnf file.
/// @return ENTROPIC_OK on success.
///         - ENTROPIC_ERROR_INVALID_CONFIG if key already exists
///         - ENTROPIC_ERROR_LOAD_FAILED if file unreadable
///
/// @version 1.9.3
ENTROPIC_EXPORT entropic_error_t entropic_grammar_register_file(
    entropic_handle_t handle,
    const char* key,
    const char* path);

/// @brief Remove a grammar from the registry.
/// @param handle Engine handle.
/// @param key Grammar name.
/// @return ENTROPIC_OK on success. ENTROPIC_ERROR_NOT_FOUND if key missing.
///
/// @version 1.9.3
ENTROPIC_EXPORT entropic_error_t entropic_grammar_deregister(
    entropic_handle_t handle,
    const char* key);

/// @brief Get grammar content by key.
/// @param handle Engine handle.
/// @param key Grammar name.
/// @return GBNF content string. Caller frees with entropic_free().
///         NULL if key not found.
///
/// @version 1.9.3
ENTROPIC_EXPORT char* entropic_grammar_get(
    entropic_handle_t handle,
    const char* key);

/// @brief Validate a GBNF grammar string without registering.
/// @param gbnf_content Raw GBNF string to validate.
/// @return NULL on success (valid grammar). Error description string on
///         failure. Caller frees with entropic_free().
///
/// @note This is a static function -- does not require an engine handle.
///       Can be called before engine initialization.
///
/// @version 1.9.3
ENTROPIC_EXPORT char* entropic_grammar_validate(const char* gbnf_content);

/// @brief List all registered grammars as JSON array.
/// @param handle Engine handle.
/// @return JSON array of grammar metadata objects:
///         [{"key": "compactor", "source": "bundled", "validated": true}, ...]
///         Content strings are NOT included (use entropic_grammar_get for content).
///         Caller frees with entropic_free().
///
/// @version 1.9.3
ENTROPIC_EXPORT char* entropic_grammar_list(entropic_handle_t handle);
```

## Cross-Model Grammar Chaining

The key novel capability: model A generates structured output that defines
grammar constraints for model B.

**Sequence:**

```
Model A (e.g., lead)          GrammarRegistry          Model B (e.g., eng)
        │                           │                         │
        │ generate(prompt,          │                         │
        │   grammar_key="json")     │                         │
        │◄──────────────────────────┤ resolve "json"          │
        │                           │                         │
        │ output: {"format":        │                         │
        │   "root ::= ..."}        │                         │
        │──────────────────────────►│                         │
        │ register_grammar(         │                         │
        │   "dynamic_eng",          │                         │
        │   output.format,          │                         │
        │   source="dynamic")       │                         │
        │                           │──── validate ──►        │
        │                           │                         │
        │                           │          generate(      │
        │                           │            prompt,      │
        │                           │◄── grammar_key=         │
        │                           │    "dynamic_eng")       │
        │                           │                         │
        │                           │  resolve "dynamic_eng" ─►│
        │                           │                         │
        │                           │          constrained    │
        │                           │◄──── output ────────────│
```

This is NOT a new subsystem. It composes existing pieces:
1. Generate with a grammar (existing) → model A produces structured output
2. Parse output and extract grammar content (consumer logic, not engine logic)
3. Register extracted grammar via C API (new in v1.9.3)
4. Generate with the new grammar key (new grammar_key resolution)

The engine provides the registry and resolution. The orchestration logic
(deciding which model's output to use as another model's grammar) lives in
the agentic loop or consumer code, not in the registry itself.

## Identity Frontmatter Migration

Current Python frontmatter:
```yaml
grammar: compactor.gbnf
```

After v1.9.3, frontmatter `grammar:` is resolved as a registry key:
- If the value has a `.gbnf` extension, strip it to get the key
  (`compactor.gbnf` → key `"compactor"`)
- If the value has no extension, use it as-is as the key
- This maintains backward compatibility with existing identity files

The `_resolve_grammar()` method in the orchestrator is replaced by:
1. Extract `grammar:` from identity frontmatter
2. Normalize to registry key (strip `.gbnf` if present)
3. Look up in `GrammarRegistry`
4. Pass content string to backend

Config override (`tiers.*.grammar:`) follows the same resolution.

## Orchestrator Integration

`ModelOrchestrator` gains a `GrammarRegistry` member. Grammar resolution
moves from ad-hoc file loading to registry lookup.

### Startup Sequence

```
orchestrator.initialize(config)
    │
    ├── Load models (existing v1.8.2)
    ├── Load adapters (existing v1.9.2)
    │
    ├── grammar_registry_.load_bundled(data_dir / "grammars")
    │   ├── Scan data/grammars/*.gbnf
    │   ├── Register each: key = stem, source = "bundled"
    │   ├── Validate each on registration
    │   └── Log: "Loaded N bundled grammars"
    │
    └── Ready for generation
```

### Per-Generation Resolution

```
orchestrator.generate(tier, messages, params)
    │
    ├── params.grammar non-empty?
    │   └── Yes → use raw string directly (skip registry)
    │
    ├── params.grammar_key non-empty?
    │   └── Yes → grammar_registry_.get(key) → set params.grammar
    │        └── Key not found? → log WARNING, generate unconstrained
    │
    ├── Neither set? → check identity frontmatter
    │   └── frontmatter.grammar present?
    │       └── Yes → normalize key → grammar_registry_.get(key)
    │
    └── Pass resolved params.grammar to backend.generate()
```

## Validation Implementation

GBNF validation uses llama.cpp's grammar parser. The grammar is parsed
without a model loaded -- this is a syntax check only.

```cpp
// Static method -- no engine state needed
std::string GrammarRegistry::validate(const std::string& gbnf_content) {
    // llama_sampler_init_grammar() parses GBNF and returns nullptr on failure.
    // We create a throwaway sampler to test validity.
    // llama_sampler_free() releases it immediately.
    //
    // Alternative: use llama_grammar_init() directly if exposed in the
    // version of llama.cpp we're linking. The sampler approach works
    // regardless of llama.cpp version.
}
```

The exact implementation depends on which llama.cpp grammar parsing API
is available. The key contract: validate takes a string, returns empty on
success or an error description on failure. No side effects.

## Validation Criteria

### V1: Bundled grammars loaded at startup

Initialize the engine with bundled grammars in `data/grammars/`. Verify
`grammar_registry.size()` matches the number of `.gbnf` files. Verify
`has("compactor")`, `has("scribe")`, `has("benchmark_judge")` all return true.

### V2: Get returns correct content

Register a grammar with known content. `get("key")` returns the exact GBNF
string. `get("nonexistent")` returns empty string.

### V3: Register from in-memory string

Call `entropic_grammar_register(handle, "test", gbnf_string)`. Verify
`entropic_grammar_get(handle, "test")` returns the string. Verify
`entropic_grammar_list(handle)` includes "test" with `source: "runtime"`.

### V4: Register from file

Call `entropic_grammar_register_file(handle, "chess", "/path/to/chess.gbnf")`.
Verify content matches file contents. Verify `source: "file"` in entry metadata.

### V5: Deregister removes grammar

Register a grammar, verify `has()` returns true. Deregister it, verify
`has()` returns false. Verify `get()` returns empty string.

### V6: Duplicate key rejected

Register a grammar with key "test". Attempt to register another grammar with
key "test". Verify second registration returns false / `ENTROPIC_ERROR_INVALID_CONFIG`.

### V7: Validation catches malformed GBNF

Call `entropic_grammar_validate("root ::= invalid[")`. Verify non-NULL error
string is returned. Call `entropic_grammar_validate("root ::= \"hello\"")`.
Verify NULL is returned (valid grammar).

### V8: Validation on registration

Register a grammar with invalid GBNF content. Verify `entry.validated` is
false and `entry.error` is non-empty. Grammar is still retrievable via `get()`.

### V9: grammar_key resolution in generation

Set `params.grammar_key = "compactor"`. Call generate. Verify the generation
output is constrained by the compactor grammar (valid JSON matching the
compactor schema).

### V10: Raw grammar string takes precedence over grammar_key

Set both `params.grammar = "<raw gbnf>"` and `params.grammar_key = "compactor"`.
Generate. Verify the raw grammar is used, not the compactor grammar.

### V11: Identity frontmatter grammar resolution

Configure an identity with `grammar: compactor.gbnf`. Generate with that
identity. Verify the compactor grammar is applied. Verify it resolves via
the registry (not direct file load).

### V12: Frontmatter key normalization

Configure identity with `grammar: compactor.gbnf` (with extension). Verify
it resolves to registry key `"compactor"`. Configure identity with
`grammar: compactor` (no extension). Verify same resolution.

### V13: Runtime grammar for cross-model chaining

Register a grammar with `source="dynamic"`. Use it as `grammar_key` in a
subsequent generation. Verify generation is constrained. Deregister the
dynamic grammar. Verify next generation with same key is unconstrained
(with WARNING log).

### V14: Thread safety on concurrent register/get

From thread A, register grammars in a loop. From thread B, call `get()` in
a loop. Verify no data races (ThreadSanitizer clean).

### V15: SLOC and complexity

All functions pass knots quality gates: cognitive complexity <= 15,
SLOC <= 50, returns <= 3.

### V16: Doxygen complete

Every public symbol in `grammar_registry.h` and updated type headers has
`@brief`, `@param`, `@return`, `@version`.

## Unit Tests

### test_grammar_registry.cpp

```
SCENARIO: Load bundled grammars from directory
  GIVEN a directory with 3 .gbnf files (compactor, scribe, benchmark_judge)
  WHEN load_bundled(dir) is called
  THEN it returns 3
  AND has("compactor") returns true
  AND has("scribe") returns true
  AND has("benchmark_judge") returns true
  AND all entries have source "bundled"

SCENARIO: Register grammar from in-memory string
  GIVEN a valid GBNF string
  WHEN register_grammar("test", gbnf, "runtime") is called
  THEN it returns true
  AND get("test") returns the GBNF string
  AND entry("test").source == "runtime"
  AND entry("test").validated == true

SCENARIO: Register grammar from file
  GIVEN a .gbnf file on disk
  WHEN register_from_file("chess", path) is called
  THEN it returns true
  AND get("chess") matches file contents
  AND entry("chess").source == "file"

SCENARIO: Register with empty key uses filename stem
  GIVEN a .gbnf file named "custom_format.gbnf"
  WHEN register_from_file("", path) is called
  THEN has("custom_format") returns true

SCENARIO: Duplicate key registration fails
  GIVEN "test" is already registered
  WHEN register_grammar("test", different_gbnf) is called
  THEN it returns false
  AND get("test") returns the original content

SCENARIO: Deregister removes grammar
  GIVEN "test" is registered
  WHEN deregister("test") is called
  THEN it returns true
  AND has("test") returns false
  AND get("test") returns ""

SCENARIO: Deregister nonexistent key returns false
  WHEN deregister("nonexistent") is called
  THEN it returns false

SCENARIO: Validate accepts well-formed GBNF
  GIVEN a valid GBNF string: 'root ::= "hello"'
  WHEN validate(gbnf) is called
  THEN it returns "" (empty = success)

SCENARIO: Validate rejects malformed GBNF
  GIVEN an invalid GBNF string: 'root ::= invalid['
  WHEN validate(gbnf) is called
  THEN it returns a non-empty error description

SCENARIO: Invalid grammar registered with error metadata
  GIVEN an invalid GBNF string
  WHEN register_grammar("bad", invalid_gbnf) is called
  THEN it returns true (grammar is stored)
  AND entry("bad").validated == false
  AND entry("bad").error is non-empty
  AND get("bad") returns the invalid string

SCENARIO: List returns all registered grammars
  GIVEN 3 grammars registered
  WHEN list() is called
  THEN it returns 3 entries with correct keys and sources

SCENARIO: Clear removes all grammars
  GIVEN 3 grammars registered
  WHEN clear() is called
  THEN size() returns 0

SCENARIO: Load bundled skips invalid files with warning
  GIVEN a directory with 2 valid .gbnf files and 1 invalid
  WHEN load_bundled(dir) is called
  THEN it returns 3 (all loaded, invalid one has error metadata)
  AND the invalid entry has validated == false
```
